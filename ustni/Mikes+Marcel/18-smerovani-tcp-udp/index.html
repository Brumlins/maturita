Směrování a transportní vrstva

📌 Klíčové body

  • Statické vs dynamické směrování (výhody/nevýhody)
  • RIP (max 15 hopů), OSPF (Dijkstra, cost), BGP (internet)
  • TCP: spojovaný, spolehlivý, 3-way handshake
  • UDP: rychlý, bez záruky; porty 0–1023 well-known

Směrování – statické vs. dynamické

Statické směrování

Admin ručně zadá záznamy do směrovací tabulky. ✓ Jednoduché, bezpečné, deterministické. ✗ Neadaptuje se na výpadky, nepraktické ve velkých sítích.

ip route add 10.0.2.0/24 via 192.168.1.1
# nebo
ip route add default via 192.168.1.1

Dynamické směrování

Routery si vyměňují informace o sítích automaticky. ✓ Adaptuje se, škáluje. ✗ Složitější konfigurace, využívá CPU/RAM.

TCP 3-way handshake

Klient                    Server
  |---  SYN (seq=x)   --->|   # Chci se spojit
  |<-- SYN-ACK (seq=y)  --|   # OK, potvrzuji, mám svůj seq
  |--- ACK (seq=x+1)  --->|   # Potvrzuji tvůj seq
  |==== DATA přenos  =====|
  |--- FIN             --->|   # Chci se odpojit
  |<-- ACK               --|
  |<-- FIN               --|
  |--- ACK             --->|
