Základní TCP/IP protokoly

📌 Klíčové body

  • DHCP DORA: Discover → Offer → Request → Ack
  • ARP: IP → MAC (broadcast request, unicast reply)
  • DNS záznamy: A, AAAA, MX, CNAME, PTR, NS, TXT
  • HTTP metody: GET/POST/PUT/DELETE; SMTP/POP3/IMAP

DHCP – Dynamic Host Configuration Protocol

DHCP automaticky přiděluje IP adresu, masku, výchozí bránu a DNS server klientům v síti. UDP port 67 (server), 68 (klient).

DORA – 4 kroky DHCP

KrokZprávaKdoPopis
1DiscoverKlient → BroadcastHledám DHCP server (src: 0.0.0.0, dst: 255.255.255.255)
2OfferServer → KlientNabízím IP adresu a konfiguraci
3RequestKlient → BroadcastPřijímám nabídku (broadcast – mohou být 2 servery)
4AcknowledgeServer → KlientPotvrzuji, konfigurace platí (lease time)

ARP – Address Resolution Protocol

ARP překládá IP adresu → MAC adresu. Funguje na L2/L3 hranici v rámci LAN. Data se na síti posílají podle MAC adres! 
# Krok 1: ARP Request (broadcast)
  Kdo má IP 192.168.1.10? Řekni mi (moje MAC je AA:BB:CC:DD:EE:FF)
  → dst MAC: FF:FF:FF:FF:FF:FF (broadcast)

# Krok 2: ARP Reply (unicast)
  Já mám 192.168.1.10, moje MAC je 11:22:33:44:55:66
  → dst MAC: AA:BB:CC:DD:EE:FF (přímá odpověď)

# ARP cache (Linux)
arp -n
ip neigh show

# Gratuitous ARP – oznámení "jsem tady" (po spuštění)
# ARP Spoofing – útok: falešné ARP odpovědi (MITM)

DNS – Domain Name System

DNS překládá doménové jméno → IP adresu. Hierarchická struktura: kořenové servery (.) → TLD (.cz, .com) → autoritativní NS. UDP/TCP port 53.
ZáznamFunkcePříklad
AJméno → IPv4seznam.cz → 77.75.79.222
AAAAJméno → IPv6google.com → 2607:f8b0::...
MXMailový server doménygmail.com → mail.google.com
CNAMEAlias → jiné jménowww.firma.cz → firma.cz
PTRIP → jméno (reverzní)77.75.79.222 → seznam.cz
NSAutoritativní DNS serverseznam.cz NS ns1.seznam.cz
TXTLibovolný text (SPF, DKIM)v=spf1 include:...
nslookup www.seznam.cz      # překlad jména na IP
nslookup -type=MX gmail.com # MX záznamy
dig seznam.cz A             # detailní DNS dotaz
host google.com

FTP – File Transfer Protocol

FTP přenáší soubory. Dva kanály: řídící (port 21) + datový (port 20 aktivní / náhodný pasivní). FTPS = FTP + TLS. SFTP = SSH-based (jiný protokol, port 22). 
ftp 192.168.1.10
sftp user@server.cz
