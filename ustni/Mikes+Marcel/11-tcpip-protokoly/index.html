Popis základních TCP/IP protokolů

11.1 TCP/IP model

TCP/IP (Transmission Control Protocol / Internet Protocol) je základní síťový protokolový model používaný v internetu a počítačových sítích.

  • TCP/IP je organizován do čtyř vrstev, kde každá má specifickou funkci:
    • Aplikační vrstva – poskytuje síťové služby (HTTP, FTP, DNS, SMTP).
    • Transportní vrstva – zajišťuje spolehlivý (TCP) nebo nespolehlivý (UDP) přenos dat.
    • Síťová vrstva – směrování dat mezi sítěmi (IP, ICMP, ARP).
    • Vrstva síťového rozhraní – připojení k fyzickému médiu (Ethernet, Wi-Fi).
  • Model TCP/IP je jednodušší než OSI model (který má 7 vrstev), ale plní podobný účel.

11.2 DHCP (Dynamic Host Configuration Protocol)

  • Co DHCP poskytuje:
    • IP adresu – dynamicky přidělenou adresu pro komunikaci v síti.
    • Masku podsítě – definuje velikost a strukturu sítě.
    • Výchozí bránu (gateway) – směřuje síťový provoz mimo lokální síť.
    • DNS servery – zajišťují překlad doménových jmen na IP adresy.
    • Volitelné informace – např. NTP servery pro synchronizaci času.
  • Hlavní fáze přidělení IP adresy pomocí DHCP:
    • DHCP DISCOVER – klient vyšle žádost o IP adresu.
    • DHCP OFFER – server nabídne dostupnou IP adresu.
    • DHCP REQUEST – klient potvrzuje nabídnutou IP adresu.
    • DHCP ACKNOWLEDGE – server potvrdí přidělení IP adresy.
  • Výhody DHCP:
    • Automatizace správy IP adres v síti.
    • Eliminace chyb způsobených ruční konfigurací.
    • Efektivní využití dostupných IP adres.

11.3 ARP (Address Resolution Protocol)

Definice: ARP slouží k zjištění MAC adresy zařízení na základě jeho IP adresy.

  • Proces ARP:
    • Zařízení A chce poslat data na zařízení B, ale nezná jeho MAC adresu.
    • Zařízení A vyšle ARP požadavek („Kdo má tuto IP adresu?") – ARP request (broadcast).
    • Zařízení B odpoví ARP odpovědí a sdělí svou MAC adresu – ARP reply (unicast).
    • Zařízení A si tuto MAC adresu uloží do ARP tabulky.
  • Ping – linka komunikuje na MAC adresách routeru (ARP).
  • Bezpečnostní hrozba: ARP spoofing
    • Útočník podvrhne svou MAC adresu do ARP tabulky oběti.
    • Zařízení v síti pak komunikují s útočníkem místo s routerem.
    • Útočník může odposlouchávat nebo manipulovat s daty.

11.4 DNS (Domain Name System)

Definice: DNS převádí doménová jména (např. www.example.com) na IP adresy (např. 192.168.1.1).

  • DNS funguje jako telefonní seznam internetu. Internet pracuje na adresách, DNS pracuje na jménech.
  • Hlavní součásti DNS systému:
    • DNS Resolver – přijímá požadavky na překlad doménových jmen. Zjistí jméno za IP adresu, zeptá se kořenového serveru, poté TLD serveru (např. .cz), poté autoritativního serveru (např. seznam.cz).
    • DNS servery – poskytují odpovědi na DNS dotazy.
    • DNS zóny – ukládají záznamy o doménách.
  • Kořenová doména → TLD doména (2. třída) → konkrétní doména.
  • DNS databáze – záznamy:
    • A – IP adresa (IPv4).
    • AAAA – IPv6 adresa.
    • MX – mailový server domény.
  • DNS databáze – praxe:
    nslookup www.seznam.cz → 77.75.79.222

11.5 HTTP a HTTPS (Hypertext Transfer Protocol / Secure)

Definice: HTTP je protokol používaný pro přenos webových stránek a dat mezi klientem a serverem.

  • Základní vlastnosti HTTP:
    • Slouží k výměně dokumentů ve formátu HTML.
    • Používá princip dotaz-odpověď mezi klientem (webovým prohlížečem) a serverem.
    • Je bezstavový – každá komunikace je nezávislá, což řeší cookies nebo sessions.
  • HTTPS zajišťuje bezpečnost přenosu pomocí šifrování:
    • SSL/TLS – zabezpečuje komunikaci mezi klientem a serverem.
    • Autentizace serveru – ověření, že komunikujete se správným serverem.
    • Integrita dat – ochrana proti neoprávněné manipulaci s daty.
  • Základní metody HTTP:
    • GET – žádost o zobrazení zdroje (webové stránky).
    • POST – odeslání dat na server (např. formulář).
    • PUT – nahrazení existujícího zdroje novými daty.
    • DELETE – odstranění zdroje.
    • HEAD – vrací pouze hlavičky, ne obsah stránky.

11.6 POP3, SMTP a IMAP (E-mailové protokoly)

  • POP3 (Post Office Protocol 3)
    • Slouží k stahování e-mailů z e-mailového serveru na místní zařízení.
    • Po stažení jsou e-maily obvykle odstraněny ze serveru (přesune mailbox na program uživatele, na serveru to už nebude).
    • Umožňuje čtení e-mailů offline.
  • SMTP (Simple Mail Transfer Protocol)
    • Používá se pro odesílání e-mailů mezi servery.
    • E-mailový klient odesílá zprávu na SMTP server, který ji doručí příjemci.
    • Odeslat do mailboxu a přeposílat uživateli.
  • IMAP (Internet Message Access Protocol)
    • Umožňuje správu e-mailů přímo na serveru.
    • Uživatel může organizovat e-maily bez jejich stahování (zůstane na serveru).
    • Synchronizuje e-maily na více zařízeních.

11.7 FTP (File Transfer Protocol)

Definice: FTP je platformně nezávislý protokol pro přenos souborů mezi klientem a serverem. Zasílání a výměna souborů.

  • Používá dva porty:
    • TCP/21 – ovládací kanál pro příkazy.
    • TCP/20 – datový kanál pro přenos souborů.
  • Hlavní vlastnosti FTP:
    • Podporuje textový i binární přenos dat.
    • Lze jej používat přes webové prohlížeče nebo specializované FTP klienty.
    • Vyžaduje uživatelské jméno a heslo pro přihlášení.
  • Bezpečnostní nevýhody:
    • Přihlašovací údaje i data jsou přenášena nešifrovaně.
    • Zabezpečené varianty:
      • FTPS (FTP Secure) – používá SSL/TLS pro šifrování.
      • SFTP (Secure FTP) – běží nad SSH protokolem.
  • Režimy přenosu dat:
    • Pasivní režim – klient zahajuje spojení pro přenos dat (užitečné v sítích s NAT).
    • Aktivní režim – server zahajuje spojení, klient pouze naslouchá.
