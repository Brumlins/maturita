Struktura souborového systému

📌 Klíčové body

  • FHS – Filesystem Hierarchy Standard
  • Klíčové adresáře: /etc, /var, /proc, /dev, /home
  • /etc/fstab – automatické připojení oddílů
  • Inode – co ukládá a co ne, hard vs symlink

Adresářová struktura Linuxu (FHS)

FHS = Filesystem Hierarchy Standard – popisuje, kde mají být jaké soubory v Linuxu.
AdresářObsah
/Kořen – vše je pod ním
/binZákladní příkazy (ls, cp, bash) – dostupné i bez /usr
/sbinSystémové příkazy pro roota (fsck, ip, mount)
/etcKonfigurační soubory (/etc/passwd, /etc/fstab, /etc/hostname)
/homeDomovské adresáře uživatelů (/home/jan)
/rootDomovský adresář roota
/varProměnná data – logy (/var/log), databáze, mail spools
/tmpDočasné soubory – mažou se při restartu
/usrUživatelské programy, knihovny (/usr/bin, /usr/lib)
/procVirtuální FS – informace o procesech (PID, CPU, paměť)
/sysVirtuální FS – informace o HW, ovladačích
/devZařízení jako soubory (/dev/sda, /dev/null, /dev/tty)
/bootBootloader, jádro (vmlinuz, initrd)
/mnt, /mediaPřípojné body pro external FS

Oddíly a /etc/fstab

# Zobrazit tabulku oddílů
lsblk
fdisk -l

# /etc/fstab – automatické připojení při startu
# DEVICE         MOUNTPOINT    TYPE    OPTIONS       DUMP  PASS
UUID=abc-123     /             ext4    defaults      0     1
UUID=def-456     /home         ext4    defaults      0     2
UUID=ghi-789     swap          swap    sw            0     0
tmpfs            /tmp          tmpfs   nosuid,nodev  0     0

Inody (Inode)

Inode = datová struktura na disku. Ukládá vše o souboru KROMĚ jména:
UID, GID, oprávnění, časy (atime, mtime, ctime), počet odkazů, ukazatele na datové bloky.
Jméno souboru je uloženo v adresáři jako dvojice: (jméno → číslo inodu).
Hard link = dvě jména sdílí jeden inode. Symlink = soubor s cestou. 
# Zobrazit inode číslo
ls -li soubor.txt

# Informace o souboru (inode)
stat soubor.txt

# Počet volných inodů
df -i

Správa disků a oddílů

# Oddíly (interaktivní)
fdisk /dev/sdb
gdisk /dev/sdb          # GPT

# Alternativa: parted
parted /dev/sdb mklabel gpt
parted /dev/sdb mkpart primary ext4 0% 50%

# RAID softwarový
mdadm --create /dev/md0 --level=1 --raid-devices=2 /dev/sdb /dev/sdc

# LVM
pvcreate /dev/sdb
vgcreate myvg /dev/sdb
lvcreate -L 20G -n mylv myvg
mkfs.ext4 /dev/myvg/mylv
