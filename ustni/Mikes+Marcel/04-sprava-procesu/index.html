Správa procesů

// Klíčové pojmy

  • Proces = aktivní instance programu
  • PID, PPID, UID, nice (-20 až 19)
  • Stavy: Running, Sleeping, Zombie, Stopped
  • kill, SIGTERM (15), SIGKILL (9), fg/bg/jobs

Co je proces?

Proces = aktivní instance programu v paměti. OS mu přiděluje zdroje (CPU čas, paměť). Má vlastní adresový prostor (kód, data, zásobník, halda) a je izolovaný od ostatních.

Vztah proces – vlákno

Proces

Má vlastní adresový prostor. Izolovaný – pád jednoho neovlivní jiný. Vytváří se přes fork().

Vlákno (Thread)

Lehký proces – sdílí adresový prostor s ostatními vlákny téhož procesu. Vlastní zásobník, registry a programový čítač. Rychlejší přepínání, ale race conditions.

Parametry procesu

ParametrPopis
PIDProcess ID – unikátní číslo procesu. systemd/init má vždy PID 1.
PPIDParent PID – kdo proces vytvořil (fork)
UID/GIDUživatel a skupina, pod kterou proces běží
nicePriorita: -20 (nejvyšší) až 19 (nejnižší). Výchozí 0.
StateRunning / Ready / Sleeping / Zombie / Stopped

Stavy procesu

R Running
Právě běží na CPU
S Sleeping
Čeká na I/O nebo událost
Z Zombie
Skončil, rodič nepřečetl exit code

Signály a kill

Signál#Popis
SIGHUP1Reload konfigurace
SIGINT2Ctrl+C – přerušení
SIGKILL9Násilné zabití – NELZE zachytit
SIGTERM15Žádost o ukončení (výchozí)
SIGSTOP19Pozastavení – nelze zachytit

Příkazy pro procesy

ps aux                    # snapshot všech procesů
ps -eo pid,ppid,ni,stat,cmd
top / htop                # interaktivní monitor
pstree -p                 # strom procesů s PID

kill 1234                 # SIGTERM
kill -9 1234              # SIGKILL
killall firefox           # podle jména
pkill -u jan              # kill všech procesů uživatele

nice -n 10 ./skript.sh    # spustit s nižší prioritou
renice -n 5 -p 1234       # změnit prioritu běžícího procesu

# Popředí / pozadí
Ctrl+Z                    # pozastaví aktuální proces
bg                        # pošle na pozadí
fg                        # přenese do popředí
jobs -l                   # seznam jobů
