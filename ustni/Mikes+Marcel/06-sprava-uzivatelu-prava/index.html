Práva, ACL a adresářové služby

📌 Klíčové body

  • chmod (symbolicky i numericky), chown, chgrp
  • SUID (4), SGID (2), Sticky bit (1)
  • ACL: getfacl / setfacl – granulární oprávnění
  • LDAP (port 389/636) a Active Directory

Oprávnění v Linuxu (DAC)

Každý soubor má vlastníka (user), skupinu (group) a práva pro ostatní (other). Tři sady: čtení (r=4), zápis (w=2), spuštění (x=1). 
ls -l soubor.txt
# -rwxr-xr-- 1 jan vyvojari 1234 Jan 10 09:00 soubor.txt
#  ^^^------  vlastník=rwx, skupina=r-x, ostatní=r--

# Symbolicky
chmod u+x skript.sh          # přidat execute vlastníkovi
chmod go-w soubor.txt        # odebrat write skupině a ostatním
chmod a=r soubor.txt         # nastavit jen read pro všechny

# Numericky (octal)
chmod 755 skript.sh          # rwxr-xr-x
chmod 644 soubor.txt         # rw-r--r--
chmod 600 /etc/secret        # rw------- (jen vlastník)
chmod 700 ~/privatni/        # rwx------ (adresář jen vlastník)

# Vlastník a skupina
chown jan soubor.txt
chown jan:vyvojari soubor.txt
chgrp vyvojari soubor.txt
chown -R jan:jan /var/www/   # rekurzivně

Speciální bity

BitOctalNa souboruNa adresáři
SUID4000Spustí se jako vlastník (passwd, sudo)Bez efektu
SGID2000Spustí se jako skupinaNové soubory dědí skupinu adresáře
Sticky1000—Mazat může jen vlastník (/tmp)
chmod u+s /usr/bin/passwd    # SUID
chmod g+s /var/shared/       # SGID na adresáři
chmod +t  /tmp               # Sticky bit
chmod 4755 skript.sh         # SUID + rwxr-xr-x

ACL – Access Control Lists

Standardní Unix oprávnění umožňují nastavit práva pouze pro jednoho uživatele, jednu skupinu a ostatní. ACL umožňují granulární nastavení pro libovolné množství uživatelů a skupin. 
# Zobrazit ACL
getfacl soubor.txt

# Nastavit ACL pro uživatele
setfacl -m u:jana:rw soubor.txt      # jana může číst a psát
setfacl -m g:audit:r soubor.txt      # skupina audit jen číst
setfacl -m o::--- soubor.txt         # ostatní vůbec nic

# Výchozí ACL (dědí nové soubory v adresáři)
setfacl -d -m u:jana:rw /var/shared/

# Odebrat ACL
setfacl -x u:jana soubor.txt
setfacl -b soubor.txt                # smaž vše ACL

# Výstup getfacl:
# user::rw-
# user:jana:rw-
# group::r--
# mask::rw-
# other::---

LDAP a Active Directory

LDAP

Lightweight Directory Access Protocol – protokol pro přístup k adresářovým službám. Hierarchická struktura (DC, OU, CN). Port 389 (plain), 636 (LDAPS).

Active Directory (AD)

Microsoft implementace LDAP + Kerberos. Domény, OU, GPO (Group Policy Objects). Centrální správa uživatelů Windows sítě. Na Linuxu lze připojit přes SSSD nebo realmd.
