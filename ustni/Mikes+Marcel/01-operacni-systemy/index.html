Operační systémy

1.1 Vývoj operačních systémů

Historie OS (1940–1990)

1940–1950
Počátky – OS pro sálové počítače, dávkové zpracování úloh.
1960–1970
Dávkové zpracování – zavedeno multiprogramování, umožnění běhu více úloh.
1970–1980
Víceúlohové systémy – zavedení Unixu, multitasking a správa paměti.
1980–1990
Grafická rozhraní – vznik GUI, rozvoj osobních počítačů.

Moderní OS (1990–současnost)

1990–2000
Otevřené a uzavřené OS – Linux (otevřený) vs. Windows (uzavřený).
2000–2010
Virtualizace a mobilní OS – virtualizace, příchod Androidu a iOS.
2010–současnost
Cloud a zabezpečení – cloudová integrace, zvýšený důraz na bezpečnost.

1.2 Struktura OS

Operační systém je prostředí mezi hardwarem a softwarem. Spravuje: procesor, paměť, externí paměť (filesystem), uživatele, vstupně/výstupní zařízení.

  • Jádro (Kernel) – spravuje hardware a zdroje, může být monolitické (Linux) nebo mikrojádro (Minix).
  • Správa procesorů – alokace procesorového času procesům.
  • Procesy a vlákna – multitasking, dělení na procesy a lehčí vlákna.
  • Systémová volání – rozhraní mezi aplikacemi a jádrem OS.

1.2.1 Funkce operačního systému

  • Základ pro práci OS je hardware (počítač)
  • Přebírá pod svoji kontrolu veškeré části počítače
  • Přiděluje procesům (= spuštěný program) část paměti, kterou mohou používat pro čtení/zápis
  • Přiděluje procesům periférie (tiskárny, skenery, disk, atd.)
  • Má pod kontrolou procesor (přiděluje část procesoru procesům)
  • Stará se o systém souborů, vytváří a ruší soubory a adresáře

1.3 Typy OS

  • Mono-uživatelský – určen pro jednoho uživatele (Windows, macOS).
  • Víceuživatelské víceúlohové (multi-user multi-task) – možnost zpracovávat požadavky více uživatelů přihlášených do systému.
  • Diskový OS – spravuje soubory na discích (NTFS, FAT32, ext4).
  • Síťový OS – správa síťových zdrojů a komunikace (Windows Server, Novell NetWare).
  • Virtualizační OS – umožňuje běh více virtuálních strojů (VMware, Hyper-V).
  • Desktop – slouží uživateli „na stole" (osobní počítač), cílem je nabídnout jednoduchost, práci s mnoha aplikacemi.
  • Otevřený operační systém – otevřené operační systémy mají veřejně dostupný zdrojový kód (Linux – Fedora, OpenSUSE, Linux Mint).
  • Server – obsluhují uživatele internetu a provádějí složité výpočty, cílem je nabídnout vysoký výpočetní výkon, snadnou údržbu, odolnost proti útokům, možnost úpravy systému.
  • Jednouživatelské jednoúlohové (single-user single-task) – bez ochrany souborů a disků, neumožňuje běh více procesů najednou.
  • Jednouživatelské víceúlohové (single-user multi-task) – multitasking, možnost běhu více procesů najednou.

1.4 CLI / GUI rozdíl a využití

CLI (Command Line Interface)

  • Textové rozhraní
  • Využívá terminálu
  • Efektivní pro pokročilé uživatele nebo administrátory

Výhody:

  • Efektivita, rychlost
  • Automatizace pomocí skriptů
  • Menší nároky na systémové prostředky

Nevýhody:

  • Obtížnější pro začátečníky
  • Méně intuitivní než GUI
  • Vyžaduje znalost příkazů a syntaxe

GUI (Graphical User Interface)

  • Grafické rozhraní

Výhody:

  • Vizualizace dat a informací
  • Intuitivní pro začátečníky

Nevýhody:

  • Vyšší nároky na systémové prostředky
  • Méně efektivní
  • Omezená

1.5 Proces zavedení a spuštění OS

  1. POST (Power-On Self Test) – kontrola hardware.
  2. Načtení zaváděcího sektoru – bootloader načte jádro OS.
  3. Zavedení jádra OS – inicializace systémových zdrojů.
  4. Spuštění systémových procesů – aktivace základních služeb OS.
  5. Aktivace uživatelského rozhraní – GUI nebo CLI pro interakci s uživatelem.

1.6 Logy v operačním systému

Definice: Logy jsou záznamy událostí, slouží k diagnostice, sledování výkonu a zabezpečení.

Typy logů:

  • Systémové logy – informace o chodu OS.
  • Aplikační logy – chování jednotlivých aplikací.
  • Bezpečnostní logy – pokusy o přístup, ochrana proti útokům.
  • Transakční logy – změny v databázích.
  • Logy o výkonu – sledování systémových zdrojů.

Logování v Linuxu a Windows

  • Linux – logy v /var/log/syslog, nástroj syslog.
    • Příkaz: tail -f /var/log/syslog
  • Windows – Prohlížeč událostí (eventvwr.msc), logy aplikací, systému a zabezpečení.
