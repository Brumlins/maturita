Služby

5.1 Význam

Služba je speciální typ programu, který běží na pozadí operačního systému bez přímé interakce s uživatelem. Služby mohou být automaticky spouštěny při startu systému a zajišťují důležité funkce, jako je správa sítí, databází nebo zabezpečení. Nabízejí určité funkce nebo zdroje nezávisle na konkrétním uživateli (fungují i bez přihlášení uživatele).

5.1.1 Hlavní funkce služeb

  • Běží na pozadí operačního systému.
  • Dají se spouštět automaticky při startu OS.
  • Poskytují klíčové funkce pro chod systému (například tisková služba, ovladače).

5.2 Ovládání služeb – zastavení, spuštění, povolení a zakázání

Systemctl je centrální nástroj pro správu služeb a jejich stavu pro Linux systémy používající systemd. Pomocí něj můžeme kontrolovat stav služby, její nastavení a měnit chování.

Správa služeb v Linuxu (systemd):

  • Zastavení služby: systemctl stop [název_služby]
  • Spuštění služby: systemctl start [název_služby]
  • Restart služby: systemctl restart [název_služby]
  • Znovunačtení: systemctl reload [název_služby]
  • Kontrola stavu služby: systemctl status [název_služby]
  • Povolení služby při startu: systemctl enable [název_služby]
  • Zakázání služby při startu: systemctl disable [název_služby]
  • Seznam aktivních služeb: systemctl list-units --type=service

Správa služeb ve Windows (Services.msc a příkazový řádek):

  • Spuštění služby: sc start [název_služby]
  • Zastavení služby: sc stop [název_služby]
  • Kontrola stavu služby: sc query [název_služby]
  • Povolení služby při startu: sc config [název_služby] start= auto
  • Zakázání služby při startu: sc config [název_služby] start= disabled

5.3 Příklady služeb běžících v PC

Služby na serveru:

  • Webový server: Apache, Nginx.
  • Databázový server: MySQL – správa dat pro aplikace.
  • Síťové služby: SSH (sshd) – vzdálená správa serveru.
  • Monitoring: Nagios, Prometheus – sledují stav systému.

Služby na desktopovém systému (linux):

  • Tiskové služby: CUPS – správa tiskových úloh. (Windows: Printing Service)
  • Síťové služby: NetworkManager – správa síťových připojení.
  • Zabezpečení: Firewalld, uwf – správa firewallu.
  • Sdílení: Samba, NFS.

5.4 Plánovač úloh v Linuxu

CRON je plánovací nástroj v Linuxu, který umožňuje automatické spouštění úloh ve specifických časových intervalech.

  • Uživatelský cron: Každý uživatel může naplánovat úlohy (např. zálohování dat).
  • Administratorský cron: Spravuje systémové úlohy (např. aktualizace, čištění logů).

Příklady zápisu do crontabu:

* * * * * command = Min (0-59) | H (0-23) | Dm (month 1-31) | M (1-12) | Dw (week 1-7)

  • Denní záloha ve 22:00: 0 22 * * * /usr/bin/backup.sh
  • Aktualizace každé pondělí ve 3:00 (Debian): 0 3 * * 1 apt update -y && apt upgrade -y
  • Čištění logů každou neděli ve 2:00: 0 2 * * 7 /usr/bin/logrotate

Plánování úloh ve Windows (Task Scheduler):

  • Umožňuje nastavit úlohy na základě času, událostí nebo podmínek.
  • Lze použít grafické rozhraní nebo příkazový řádek.

5.5 Instalace softwaru v Linuxu

Instalace softwaru v Linuxu probíhá přes balíčkovací systémy, které spravují závislosti a zajišťují snadnou instalaci, aktualizaci a odstranění programů.

Nejčastější balíčkovací systémy:

DistribuceBalíčkovací systémPříkaz
Debian/UbuntuAPTsudo apt install [balíček]
FedoraDNFsudo dnf install [balíček]

Další užitečné příkazy:

  • Aktualizace repozitářů: sudo apt update
  • Aktualizace nainstalovaného softwaru: sudo apt upgrade
  • Odstranění programu: sudo apt remove [balíček]
