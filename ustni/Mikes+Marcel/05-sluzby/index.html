Služby

// Klíčové pojmy

  • Daemon = program na pozadí
  • systemctl start/stop/enable/disable/status
  • Unit soubory: [Unit], [Service], [Install]
  • crontab – minuta hodina den měsíc den_týdne

Co je služba?

Služba (daemon) = program běžící na pozadí bez interakce s uživatelem. Spouštěna automaticky při startu OS. Zajišťuje funkce sítě, databáze, tisku, bezpečnosti atd.

systemd – správa služeb

systemd = moderní init systém Linuxu (PID 1). Nahradil SysV init. Paralelní start, závislosti, žurnálování. 
# Ovládání
systemctl start   nginx
systemctl stop    nginx
systemctl restart nginx
systemctl reload  nginx       # reload konfigurace bez restartu
systemctl status  nginx       # stav + posledních pár řádků logu

# Autostart při bootu
systemctl enable  nginx
systemctl disable nginx
systemctl mask    nginx       # zablokuj (nelze spustit)

# Přehled
systemctl list-units --type=service --state=running
systemctl list-unit-files --type=service

Unit soubor (.service)

# /etc/systemd/system/mojesluzba.service
[Unit]
Description=Moje vlastní služba
After=network.target

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/app
ExecStart=/opt/app/server
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
systemctl daemon-reload          # načti nové unit soubory
systemctl enable --now mojesluzba

Logy – journald

journalctl -u nginx              # logy konkrétní služby
journalctl -f                    # live sledování (follow)
journalctl -n 50                 # posledních 50 řádků
journalctl --since "1 hour ago"
journalctl -p err                # pouze chyby

CRON – plánování úloh

Crontab syntaxe: minuta hodina den_měsíce měsíc den_týdne příkaz 
crontab -e              # edituj crontab aktuálního uživatele
crontab -l              # zobraz

# Příklady
0 2 * * *   /backup.sh          # každý den ve 2:00
*/5 * * * * /check.sh           # každých 5 minut
0 2 * * 0   /weekly.sh          # každou neděli ve 2:00
0 2 1 * *   /monthly.sh         # 1. v měsíci ve 2:00

Správa balíčků – apt

apt update                  # aktualizuj seznam balíčků
apt upgrade                 # aktualizuj nainstalované
apt install nginx           # instaluj
apt remove nginx            # odinstaluj (zachová konfig)
apt purge  nginx            # odinstaluj + smaž konfig
apt autoremove              # odstraň nepotřebné závislosti
apt search php              # hledání balíčků
