Správa uživatelů

📌 Klíčové body

  • useradd, usermod, userdel, passwd
  • Klíčové soubory: /etc/passwd, /etc/shadow, /etc/group
  • su, sudo – přepínání kontextu
  • PAM – modulární autentizace

Uživatelé v Linuxu

root (UID 0)

Superuživatel – neomezená práva. Přihlášení přes sudo -i nebo su -. V moderních systémech se přímé přihlášení jako root blokuje.

Systémoví uživatelé (UID 1–999)

Spouštějí systémové služby (www-data, mysql, nobody). Nemají domovský adresář ani shell.

Klíčové soubory

SouborObsah
/etc/passwdUživatelé: jméno:x:UID:GID:GECOS:home:shell
/etc/shadowHashovaná hesla + stáří, expirace (jen root čte)
/etc/groupSkupiny: jméno:x:GID:členové
/etc/gshadowHesla skupin (vzácně používáno)
/etc/sudoersKdo smí volat sudo a pro jaké příkazy

Správa uživatelů – příkazy

# Vytvoření uživatele
useradd -m -s /bin/bash -G sudo jan     # -m = domovský adresář
adduser jan                              # interaktivní (Debian)

# Nastavení hesla
passwd jan

# Úprava uživatele
usermod -aG docker jan        # přidat do skupiny docker
usermod -s /bin/zsh jan       # změnit shell
usermod -L jan                 # zamknout účet
usermod -U jan                 # odemknout účet

# Smazání uživatele
userdel -r jan                 # -r = smaž i domovský adresář

# Skupiny
groupadd vyvojari
groupdel vyvojari
gpasswd -a jan vyvojari        # přidat uživatele do skupiny
gpasswd -d jan vyvojari        # odebrat ze skupiny

# Přepnutí uživatele
su - jan                       # přepnout na jana
sudo -u jan příkaz             # spustit jako jan

Informace o uživatelích

id jan                  # UID, GID, skupiny
whoami                  # aktuální uživatel
who / w                 # přihlášení uživatelé
last                    # historie přihlášení
lastb                   # neúspěšná přihlášení
finger jan              # detaily (pokud je nainstalován)

PAM – Pluggable Authentication Modules

PAM = modulární systém autentizace v Linuxu. Každá služba (login, SSH, sudo) má vlastní konfiguraci v /etc/pam.d/. Moduly lze kombinovat (heslo + OTP + certifikát).
