Správa procesů – základy

📌 Klíčové body

  • Proces = spuštěný program, vlastní adresový prostor
  • Stavy: Running, Ready, Waiting, Zombie, Stopped
  • PID, PPID, vlákna (thread) – sdílená paměť
  • Plánování CFS, nice hodnota (-20 až 19)

Co je proces?

Proces = spuštěný program v paměti. Má vlastní adresový prostor (kód, data, zásobník, haldu), PID, UID, stavy a prostředky. OS přiděluje CPU přes plánovač.

Stavy procesu

StavPopis
RunningPrávě běží na CPU
ReadyČeká na přidělení CPU
Waiting/SleepingČeká na I/O nebo událost
ZombieSkončil, ale rodič ještě nepřečetl exit code
StoppedPozastaven signálem SIGSTOP

PID, PPID, vlákna

PID = Process ID – jedinečné číslo procesu
PPID = Parent PID – kdo proces vytvořil
init/systemd má vždy PID 1
Vlákno (thread) = lehký proces – sdílí adresový prostor s ostatními vlákny stejného procesu. Rychlejší přepínání, ale sdílená paměť → race conditions.

Životní cyklus procesu

# Vytvoření procesu
fork()     # zkopíruje rodičovský proces
exec()     # nahradí kód novým programem
wait()     # rodič čeká na ukončení potomka
exit()     # proces se ukončí a předá exit code

Priorita a plánovač

Linuxový plánovač CFS (Completely Fair Scheduler) rozděluje CPU čas spravedlivě. Prioritu ovlivňuje hodnota nice (-20 = nejvyšší priorita, 19 = nejnižší). 
# Spustit s nižší prioritou
nice -n 10 ./my_script.sh

# Změnit prioritu běžícího procesu
renice -n 5 -p 1234        # pid 1234

# Zobrazit nice hodnoty
ps -eo pid,ni,cmd
