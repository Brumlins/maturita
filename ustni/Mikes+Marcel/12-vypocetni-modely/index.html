Výpočetní modely

12.1 Dávkové zpracování RJE (Remote Job Entry)

Dávkové zpracování je metoda, kde jsou úkoly shromažďovány a zpracovány jako celek (dávka) v určitém časovém intervalu.

  • Proces dávkového zpracování:
    • Shromáždění dat – data jsou sbírána v určitém období.
    • Zpracování – dávka je zpracována najednou bez uživatelského zásahu.
    • Výstup – výsledky jsou generovány po dokončení dávky.
  • Použití:
    • Fakturace a finanční výpočty.
    • Noční zpracování velkých objemů dat.
    • Superpočítače a vědecké výpočty.
  • Výhody / nevýhody:
    • Efektivita při zpracování velkého množství dat.
    • Není vhodné pro interaktivní úlohy.

12.2 Host / terminál

Model, kde centrální server (host) poskytuje výpočetní služby a terminály fungují pouze jako uživatelská rozhraní.

  • Použití:
    • Bankovní systémy – přístup k databázím přes jednoduché terminály.
    • Zdravotnictví – nemocniční informační systémy.
    • Call centra – pracovníci přistupují k centrálním systémům.
    • Školství – terminály v učebnách propojené s centrálním serverem.
  • Výhody:
    • Centrální správa softwaru a bezpečnosti.
    • Snížení nákladů na koncová zařízení.
  • Nevýhody:
    • Závislost na serveru – výpadek serveru znamená výpadek celé sítě.
    • Omezená flexibilita – uživatelé nemohou instalovat vlastní aplikace.

12.3 Desktop / PC

Model, kde uživatelé mají samostatné stolní počítače s plnou výpočetní kapacitou.

  • Hlavní vlastnosti:
    • Vysoký výkon – vhodné pro zpracování textu, grafiku, hry.
    • Flexibilita – možnost instalace vlastního softwaru.
    • Nízké vstupní náklady – ideální pro domácnosti a malé podniky.
  • Použití:
    • Kancelářská práce.
    • Domácí použití, internet, hraní her.

12.4 Potřeba sdílení – nutnost LAN

  • Sdílení dat a zdrojů vyžaduje lokální síť (LAN).
  • LAN podporuje počítače v rámci organizace pro efektivnější spolupráci.

12.5 File server / Workstation

Model, kde různé servery poskytují specifické služby uživatelským stanicím.

  • File server – centralizované úložiště dat, dostupné více uživatelům.
  • Workstation – uživatelská stanice pro práci s daty, používané pro náročné úlohy.
  • Použití:
    • Kancelářské prostředí se sdílenými soubory a tiskovými službami.

12.6 Klient Server

Architektura, kde klientské aplikace posílají požadavky serverovým aplikacím, které je zpracovávají.

Hlavní komponenty:

  • Klient – uživatelské zařízení (PC, mobil), které posílá požadavky.
  • Server – centrální jednotka spravující data a aplikace.
  • Použití:
    • Webové aplikace, databázové systémy, podnikové aplikace.

12.7 Peer-to-peer (P2P)

Každé zařízení v síti funguje jako klient i server, sdílí zdroje přímo bez centrálního serveru.

Charakteristiky:

  • Decentralizace – žádný centrální bod.
  • Flexibilita – snadné přidávání a odebírání zařízení.
  • Sdílení zdrojů – soubory, výpočetní výkon.
  • Použití:
    • Sdílení souborů (např. BitTorrent).
    • Blockchain technologie.

12.8 Desktop / Network-centric computing

  • Desktop computing – všechny aplikace a data jsou na lokálním PC.
  • Network-centric computing – aplikace a data jsou umístěny na serveru.

12.9 Tenký / tlustý klient

  • Tenký klient – většina výpočtů probíhá na serveru.
    • Příklady: Citrix, Windows Remote Desktop.
  • Tlustý klient – výpočty probíhají lokálně, ale využívá se přístup k síťovým zdrojům.

12.10 Webové aplikace a jejich vývoj včetně Rich Internet Applications (RIA)

  • Webové aplikace – běží ve webovém prohlížeči (např. Gmail, Google Docs).
  • RIA (Rich Internet Applications) – webové aplikace s interaktivitou blízkou desktopovým aplikacím (např. Adobe Flash, JavaFX).

12.11 Streaming aplikací

  • Aplikace se spouštějí na vzdáleném serveru a streamují se na klienta.
  • Příklad: Google Stadia, Nvidia GeForce Now.

12.12 Server-based computing

  • Výpočty probíhají na centrálním serveru.
  • Klienti zobrazují pouze výstup.
  • Použití v terminálových službách (RDP, Citrix).

12.13 Cloud computing

  • Výpočetní zdroje poskytované na vyžádání přes internet.
  • Modely nasazení:
    • Veřejný cloud – služby poskytované třetí stranou (AWS, Google Cloud).
    • Privátní cloud – provozován interně organizací.
    • Hybridní cloud – kombinace veřejného a privátního cloudu.
  • Výhody:
    • Snížení nákladů na hardware.
    • Možnost škálování výkonu dle potřeby.
  • Nevýhody:
    • Závislost na internetovém připojení.
    • Bezpečnostní rizika spojená s ukládáním dat u poskytovatele.

12.14 Virtualizace a hypervizor

Virtualizace umožňuje běh více operačních systémů na jednom fyzickém stroji.

  • Typy hypervizorů:
    • Type 1 (bare-metal) – běží přímo na hardwaru (např. VMware ESXi, Microsoft Hyper-V).
    • Type 2 (hostovaný) – běží uvnitř operačního systému (např. VirtualBox, VMware Workstation).
  • Použití:
    • Datová centra.
    • Testovací a vývojová prostředí.

12.15 IaaS, SaaS, PaaS

  • IaaS (Infrastructure as a Service) – Hardware, který máš na starost.
    • Poskytuje virtuální servery, úložiště, sítě přes internet.
    • Příklad: AWS, Azure, Google Cloud.
  • PaaS (Platform as a Service) – Hardware managuje firma, ty jen věci na něm.
    • Nabízí vývojové prostředí pro aplikace bez správy infrastruktury.
    • Příklad: Heroku, Google App Engine.
  • SaaS (Software as a Service) – Firma managuje vše, ty to jen používáš.
    • Software poskytovaný jako služba přes internet.
    • Příklad: Google Docs, Microsoft Office 365.

12.16 Shrnutí klíčových bodů

  • Výpočetní modely určují, jak jsou spravovány výpočetní zdroje.
  • Dávkové zpracování je vhodné pro hromadné úlohy bez interakce uživatele.
  • Host/terminál model centralizuje výpočetní výkon.
  • Klient/server je nejběžnější model podnikových aplikací.
  • P2P umožňuje decentralizovanou distribuci zdrojů.
  • Cloud computing poskytuje IT služby přes internet.
