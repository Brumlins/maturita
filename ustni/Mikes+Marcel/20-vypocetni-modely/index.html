Výpočetní modely a cloud

📌 Klíčové body

  • Klient/Server – centralizovaná správa, SPOF
  • P2P – každý je klient i server, BitTorrent, blockchain
  • Virtualizace: Typ 1 (bare-metal), Typ 2 (hosted), Docker
  • Cloud: IaaS / PaaS / SaaS; veřejný / privátní / hybridní

Výpočetní modely

Klient/Server

Klient = iniciuje požadavek, zobrazuje výsledek (prohlížeč, aplikace)
Server = čeká na požadavky, zpracovává, vrací odpověď (web, databáze, e-mail)
Centralizovaná správa, snadná aktualizace, SPOF (single point of failure).
Příklady:
• Webový prohlížeč ↔ Apache/Nginx
• E-mailový klient ↔ SMTP/IMAP server
• Databázová aplikace ↔ MySQL/PostgreSQL
• SSH klient ↔ SSH server

Peer-to-Peer (P2P)

Každý uzel je zároveň klientem i serverem. Žádný centrální server. Odolné proti výpadku, škáluje samo. Využití: BitTorrent, blockchain, Napster, Skype (historicky), I2P.

Virtualizace

Virtualizace = spuštění více virtuálních počítačů (VM) na jednom fyzickém hostiteli. Hypervisor spravuje sdílení HW zdrojů.

Typ 1 – bare-metal hypervisor

Běží přímo na HW (bez OS). Vyšší výkon. Příklady: VMware ESXi, Microsoft Hyper-V, KVM (Linux), Xen.

Typ 2 – hosted hypervisor

Běží jako aplikace na OS. Pomalejší (dvojitá virtualizace). Příklady: VirtualBox, VMware Workstation, Parallels.

Kontejnerizace

Kontejner = izolovaný proces sdílející jádro OS hostitele. Lehčí než VM, rychlý start. Docker = nejrozšířenější platforma. Kubernetes = orchestrace kontejnerů (škálování, balancování). 
docker run -d -p 80:80 nginx          # spusť kontejner
docker ps                              # zobraz běžící kontejnery
docker exec -it mojec bash             # shell v kontejneru
docker-compose up -d                   # více kontejnerů dle YAML

Typy cloudu

Veřejný cloud
Sdílená infrastruktura poskytovatele. Platíš za spotřebu. AWS, Azure, GCP.
Privátní cloud
Vlastní datacentrum nebo dedikované zdroje. Kontrola, compliance. OpenStack, VMware vCloud.
Hybridní cloud
Kombinace – citlivá data on-premise, ostatní ve veřejném cloudu. Nejčastější enterprise model.
