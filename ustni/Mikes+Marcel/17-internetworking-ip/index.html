Internetworking – IP adresace

📌 Klíčové body

  • IPv4 = 32 bit, privátní: 10/8, 172.16/12, 192.168/16
  • IPv6 = 128 bit, bez NAT, SLAAC auto-konfigurace
  • CIDR notace, podsítě, výpočet sítě a broadcastu
  • TTL – zabraňuje smyčkám, traceroute exploituje TTL

IPv4 – adresace

IPv4 adresa = 32 bitů zapsaných jako 4 čísla 0–255 oddělená tečkami. Součástí je maska podsítě (prefix). Celkem ~4.3 miliardy adres – vyčerpány 2011.
TřídaRozsahPrivátní rozsah (RFC 1918)
A1.0.0.0 – 126.255.255.25510.0.0.0/8
B128.0.0.0 – 191.255.255.255172.16.0.0/12
C192.0.0.0 – 223.255.255.255192.168.0.0/16
Zvláštní adresy: 127.0.0.1 = loopback, 169.254.x.x = APIPA (bez DHCP), 0.0.0.0 = neurčeno, 255.255.255.255 = broadcast.

Podsítě a CIDR

# CIDR notace: adresa/prefix
192.168.1.0/24     # maska 255.255.255.0, 254 hostů
192.168.1.0/25     # maska 255.255.255.128, 126 hostů
10.0.0.0/8         # maska 255.0.0.0, ~16 mil. hostů

# Výpočet sítě
ip addr                         # Linux
ipconfig /all                   # Windows
ipcalc 192.168.1.100/24         # kalkulačka

IPv6 – proč a jak

128 bitů = 3.4 × 10³⁸ adres. Zápis: 8 skupin po 4 hex číslicích, odděleny :
Zkrácení: :: = skupina nul, vedoucí nuly lze vynechat.
2001:db8::1 = plně 2001:0db8:0000:0000:0000:0000:0000:0001
Výhody IPv6:
✓ NAT nepotřeba – každé zařízení má veřejnou adresu
✓ Auto-konfigurace (SLAAC)
✓ Lepší podpora mobility
✓ IPsec v základu
✓ Efektivnější směrování (hierarchie)

NAT – Network Address Translation

NAT: router překládá privátní IP:port na veřejnou IP:port. Umožňuje sdílet jednu veřejnou IP pro celou LAN. Typy: SNAT (masquerade, výchozí), DNAT (port forwarding), PAT (= NAPT, překlad portů).

IP hlavička a TTL

PoleVelikostPopis
Version4 bity4 = IPv4, 6 = IPv6
IHL4 bityDélka hlavičky (min 20 bytů)
TTL8 bitůTime To Live – snižuje se na každém routeru o 1. Při 0 → packet zahozen (ICMP Time Exceeded)
Protocol8 bitů6 = TCP, 17 = UDP, 1 = ICMP
Source IP32 bitůIP odesílatele
Dest IP32 bitůIP příjemce
TTL výchozí hodnoty: Windows = 128, Linux = 64, Cisco = 255. traceroute / tracert exploituje TTL k mapování trasy.
