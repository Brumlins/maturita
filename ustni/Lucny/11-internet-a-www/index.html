Internet a WWW

11.1 Internet – principy a vývoj, adresování v internetu

Internet je celosvětová počítačová síť, která spojuje jednotlivé menší sítě (Vint Cerf a Bob Kahn). Používá se protokol TCP/IP = komunikační pravidla pro různé síťové vrstvy. Sítě jsou pospojovány hierarchicky (Páteřní síť, lokální ISP, uživatel).

  • IntrAnet – vnitřní síť.
  • ISP (Internet Service Provider) – poskytovatel internetového připojení; firma nebo organizace poskytující přístup k internetu.
  • Páteřní připojení – používá se optika.
  • Poslední míle – připojení páteřní sítě ke koncovým uživatelům; pomocí telefonní linky, kabelové TV, satelitní datové sítě, bezdrát, mobilní telefonní sítě.
  • Peering – vzájemné propojení počítačové sítě dvou telekomunikačních společností za účelem výměny datového provozu. Např. NIX.cz, Perring.cz.

Vývoj internetu

  • Vznik na konci 60. let v Americe jako vojenský experiment → síť ARPANET.
  • Během 70. a 80. let se k ARPANETu připojovaly další akademické sítě.
  • V 90. letech se zrodil Internet.
  • V roce 1993 se připojila první česká místa; vznik sítě CESNET pro české vysoké školy.

Adresování v internetu

  • Každý PC připojený k síti má IP adresu – 32bitové číslo zapisované jako 4 dekadická čísla (oktety) v rozmezí 0–255 oddělená tečkou (192.168.1.1). IP adresa má 2 části: adresa sítě a adresa hosta.
  • Celkový počet IPv4 adres: 232 (4 294 967 296); tento počet nestačí → přechod na IPv6 s 2128 adresami.
  • Adresní prostor = rozsah použitelných adres.
  • Pro označení zdrojů se používají doménová jména (lépe zapamatovatelná).
  • DNS (Domain Name System) = provádí převod IP adres na doménová jména; DNS server má port 53.
  • Příkaz nslookup – ověření DNS nastavení.
  • TLD (Top Level Domain): generické (.com, .net, .org, .edu, .gov) / národní (.cz, .sk). Domény v ČR se registrují u nic.cz.

11.2 Principy a standardy WWW, protokol HTTP

  • World Wide Web – aplikace internetového protokolu HTTP; soustava propojených hypertextových dokumentů. Vznik 1989 v CERNu v Ženevě, díky Timu Berners-Leeovi. Základem je komunikace mezi webovým klientem a serverem.
  • Standardy WWW: HTML (struktura), CSS (vzhled), JavaScript (interaktivita), REST (architektura pro webová API).
  • Hypertext – text obsahující propojení na jiné texty pomocí hypertextových odkazů (URL).
  • Klient (prohlížeč) – program, který komunikuje s uživatelem a zasílá podněty serveru. Příklady: Google Chrome, Microsoft Edge, Firefox, LYNX (textový prohlížeč).
  • Server – program, který přijímá a obsluhuje požadavky klientů. Příklady: APACHE, NGINX.

URL adresa

Slouží k přesné specifikaci umístění zdrojů informací na internetu.

http://www.seo-napoveda.cz/clanky/on-page-faktory.php?ID=12345
  • http – použitý protokol
  • seo-napoveda – doména
  • cz – top level doména
  • clanky – adresář, ve kterém je dokument umístěn
  • on-page-faktory.php – dokument
  • ID – parametr, 12345 = hodnota parametru

Protokol HTTP

  • Hyper Text Transfer Protocol, port 80; jednoduchý aplikační protokol fungující na principu request-response.
  • Request – uživatel pošle dotaz v plain textu.
  • Response – uvozena hlavičkou se stavovým kódem (200, 404, 503…) s vrácenou webovou stránkou.
  • HTTP je původně bezestavový protokol – nepamatuje si žádný předešlý údaj. Pro zachování informací se používají Cookies (malé soubory, které se dočasně uloží na klientovi).

Protokol HTTPS

Zabezpečený HTTP, port 443. Používá šifrování na vrstvě TLS, SSL. Pro komunikaci HTTPS musí nejdříve server vlastnit certifikát (zaručí, že se vlastník nevydává za někoho jiného).

Metody HTTP

  • GET – požadavek na poslání dokumentu určeného pomocí URL.
  • POST – server má přijmout data z požadavku a něco vytvořit.
  • PUT – aktualizace záznamů v databázi.
  • DELETE – požadavek na zrušení dokumentu na serveru.

Stavové kódy odpovědi serveru

  • 1xx – informační
  • 2xx – úspěch
  • 3xx – přesměrování
  • 4xx – chyba klienta
  • 5xx – chyba serveru

11.3 Funkce internetového prohlížeče

  • Záložky, oblíbené položky, historie vyhledávání, pluginy, anonymní prohlížení, homepage, prohlížeč PDF.
  • Cache – mechanismus pro ukládání statických zdrojů (obrázky, CSS, JS) do prohlížeče pro rychlejší načítání stránek.
  • Cookies – krátké textové soubory, které navštívený web odesílá do prohlížeče.
  • Sessions – dočasné uchování uživatelských dat na straně serveru (nákupní košík).
  • Local storage – trvalé uložení dat v prohlížeči; větší kapacita než cookies.
  • Bezpečnostní aspekty: XSS – útok crashem formulářů; CORS – které domény mohou sdílet zdroje.

11.4 Typy webových služeb, vyhledávací služby, SEO, analytické nástroje

Vyhledávání = internetový vyhledávač; na základě své databáze vypisuje seznam odkazů na stránky, které obsahují informace zadané uživatelem.

Vyhledávání probíhá ve 3 krocích:

  1. Prohledávání webových stránek – speciální program crawler: dostane seznam odkazů, navštíví stránky, zapamatuje si nalezené odkazy a navštíví další stránky (cyklus se opakuje).
  2. Indexování – databáze obsahující obsahy navštívených stránek se indexuje pro rychlejší prohledávání. Indexy se řadí podle relevance (váha slova, atraktivita stránky, serióznost webu, sponzorované odkazy).
  3. Poskytnutí odpovědi na dotazy uživatelů.

SEO – optimalizace webových stránek pro vyhledávače. Cílem je získat lepší pozici ve výsledcích a tím i častější prokliknutí. Hodnocení je založeno na struktuře, obsahu, metadatech a počtu odkazů.

Analytické nástroje – poskytují data o webu jako podklad pro webového analytika nebo správce webu. Google Analytics – nejpoužívanější; umožňuje sledovat návštěvnost, chování uživatelů (kam klikají), konverze, prodeje.

11.5 Trendy vývoje webu

Web 2.0 (od roku 2004)

  • Změna komunikačního modelu z one-to-one a one-to-many směrem k many-to-many.
  • Vznik uživateli garantovaného obsahu (Wikipedia).
  • Otevřená komunikace.
  • Charakteristické technologie: Wiki systémy, MashUp (využití API k vytváření nových aplikací), sociální sítě, blogy, sdílení videa (YouTube, Vimeo).

Web 3.0

  • Sémantický web – informace uloženy podle standardizovaných pravidel; usnadňuje vyhledávání a zpracování.
  • Sdílené aplikace – ukládání uživatelských dokumentů přímo na internetu a sdílení v reálném čase (GoogleDocs).
  • Přístup přes tablet, mobilní telefon, chytrá televize.
  • Větší využití videa a 3D prostředí.
  • Webové aplikace přizpůsobené dotykovému ovládání.
  • Decentralizace – Blockchain technologie (kryptoměny, smart kontrakty).
  • IoT (Internet of Things) – síť fyzických zařízení vybavených elektronikou, softwarem, senzory a síťovou konektivitou, která si navzájem vyměňují data.
