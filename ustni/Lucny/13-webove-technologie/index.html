Webové technologie

13.1 Frontend a backend webových aplikací

Frontend je část aplikace, se kterou interaguje uživatel. Technologie: HTML (struktura), CSS (vzhled), JavaScript (interaktivita). Frameworky a knihovny: React, Angular, jQuery, Vue.

Backend je část aplikace běžící na serveru – zajišťuje logiku, zpracování dat a komunikaci s databází.

  • Webové servery: Apache, NGINX, IIS.
  • Backendové jazyky: Node.js, PHP, Python, Java, C#.
  • Databáze: MySQL, MongoDB.

13.2 JavaScript, TypeScript, DOM a BOM

JavaScript – vznikl v roce 1995; interpretovaný, objektově orientovaný a událostmi řízený jazyk. Používá dynamické typování – datový typ proměnné se určí za běhu programu.

Datové typy

  • number, string, boolean, null, object, symbol, undefined

TypeScript

TypeScript je nadstavba JavaScriptu se statickým typováním – datový typ proměnné je znám při překladu.

Deklarace proměnných

  • var – funkční platnost; lze přepisovat a měnit.
  • let – bloková platnost; lze měnit hodnotu.
  • const – bloková platnost; nelze přepsat.

DOM (Document Object Model)

Objektový model HTML dokumentu. Kořenový element je html. Každý uzel má vlastnosti, metody a události. Pomocí EventListeneru lze reagovat na události (klik, stisk klávesy). Callback funkce se volá po vyvolání události.

BOM (Browser Object Model)

Objektový model prohlížeče. Objekt window reprezentuje okno prohlížeče a poskytuje přístup k jeho vlastnostem (adresní řádek, záložky, rozměry okna).

13.3 Node.js, npm, Express.js, synchronní vs asynchronní programování

Node.js – multiplatformní runtime prostředí umožňující spouštět JavaScript na backendu. Využívá engine V8 od Google.

  • npm (Node Package Manager) – správce balíčků pro Node.js.
  • Express.js – minimalistický webový framework pro Node.js.

Synchronní vs asynchronní programování

  • Synchronní – blokuje; program čeká na dokončení operace, než pokračuje dál.
  • Asynchronní – nečeká; program pokračuje dál a výsledek zpracuje, až bude k dispozici (např. stahování dat ze serveru).

Způsoby asynchronního programování

  • Callback funkce – funkce předaná jako argument, která se zavolá po dokončení operace.
  • Promises – objekt reprezentující budoucí výsledek asynchronní operace.
  • async / await – syntaktická nadstavba nad promises pro čitelnější kód.

13.4 AJAX, REST API, GraphQL, WebSockets

AJAX (Asynchronous JavaScript and XML) – technologie umožňující měnit obsah stránky bez jejího celého znovunačtení. Používá JavaScript + Fetch API. Data se přenášejí ve formátu XML nebo JSON.

Výhody AJAX

  • Nemusí se překreslovat celá stránka.
  • Plynulost uživatelského zážitku.
  • Snížení zátěže serveru.

Nevýhody AJAX

  • Nelze předat URL se změnami obsahu.
  • Uživatel nevidí indikátor načítání.
  • Textový prohlížeč LYNX AJAX nepodporuje.

REST API

Standardní architektura pro webová API. Využívá metody GET / POST / PUT / DELETE a data ve formátu JSON nebo XML.

GraphQL

Dotazovací jazyk pro API – klient si sám určí, jaká data chce získat.

WebSockets

Protokol umožňující obousměrnou real-time komunikaci mezi klientem a serverem (chat, online hry).

13.5 Knihovny, frameworky, full-stack a vývojářské nástroje

  • Knihovna – sada hotových funkcí, které lze volat (React, jQuery).
  • Framework – hotová kostra aplikace s vlastními pravidly (Angular, Django).

Full-Stack stacky

  • MERN – MongoDB + Express.js + React + Node.js.
  • MEAN – MongoDB + Express.js + Angular + Node.js.
  • Next.js – React + Server-Side Rendering (SSR).

Frontend

  • Frameworky/knihovny: React, Angular, Vue.
  • CSS frameworky: Bootstrap, Tailwind CSS.

Backend

  • Node.js: Express.js, NestJS.
  • Python: Django, Flask.
  • PHP: Laravel, Symfony.

Vývojářské nástroje

  • VS Code – editor kódu.
  • Chrome DevTools – ladění kódu, konzole, sledování síťového provozu.
  • Postman – testování API.
  • WebPack – balení (bundlování) JS a CSS souborů.
