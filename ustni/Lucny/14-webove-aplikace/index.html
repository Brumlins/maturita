Webové aplikace

14.1 Architektura webových aplikací, MVC, Django

Webová aplikace se typicky dělí do tří vrstev:

  • Prezentační vrstva – frontend; to, co vidí uživatel.
  • Aplikační logika – backend; zpracování dat a pravidla aplikace.
  • Datová vrstva – databáze; ukládání dat.

Návrhový vzor MVC

  • Model – obsahuje data a logiku aplikace.
  • View – zobrazuje data uživateli.
  • Controller – zpracovává požadavky uživatele a propojuje Model s View.

Návrhový vzor MVT (Django)

  • Model – datové struktury (mapování na DB).
  • View – zpracovává požadavky a vrací odpověď.
  • Template – HTML výstup pro uživatele.

Django

Open-source webový framework pro Python. Nabízí: mapování databáze (ORM), administrační rozhraní, bezpečnostní funkce a šablonový systém.

django-admin startproject nazev_projektu
python manage.py startapp nazev_aplikace

Klíčové soubory: settings.py, urls.py, models.py, views.py, složka templates/.

14.2 ORM, modely, migrace, routing a views

ORM (Object-Relational Mapping)

Umožňuje pracovat s databází pomocí Python objektů místo SQL dotazů. Každý model odpovídá tabulce v databázi a dědí z models.Model.

Typy polí modelu

  • CharField – textový řetězec.
  • IntegerField – celé číslo.
  • DateField – datum.

Vztahy mezi modely

  • ForeignKey – vztah 1:N.
  • ManyToManyField – vztah M:N.

Migrace

Aktualizace databáze po změně modelů. Postup:

python manage.py makemigrations
python manage.py migrate

Routing

Definuje se v souboru urls.py pomocí funkce path(). URL může obsahovat proměnné předávané do view funkce.

Views

Obsahují hlavní logiku aplikace. Přijímají HTTP požadavek a vracejí HTTP odpověď. Funkce render() vrací vykreslenou HTML šablonu.

14.3 Šablony, formuláře a regulární výrazy

Django šablony (Templates)

  • {{ promenna }} – výpis proměnné.
  • {% for %} ... {% endfor %} – cyklus.
  • {% extends "base.html" %} – dědičnost šablon.

Formuláře

  • forms.Form – obecný formulář.
  • forms.ModelForm – formulář navázaný na model.
  • Metoda clean() slouží k validaci dat formuláře.

Regulární výrazy

  • ^ – začátek řetězce.
  • $ – konec řetězce.
  • \d – číslice.
  • + – jeden nebo více výskytů.
  • * – nula nebo více výskytů.

14.4 Autentizace, autorizace a bezpečnost

Autentizace (ověření identity)

  • Session-based – server ukládá session, klientovi posílá cookie.
  • Token-based – OAuth, JWT; token je uložen na klientovi.
  • Django: model User, funkce login() a logout().

Autorizace (kontrola oprávnění)

  • Role-based – uživatel, administrátor.
  • Permission-based – čtení, zápis.
  • Policy-based – podmínky (věk, čas).

Bezpečnostní hrozby

  • XSS (Cross-Site Scripting) – vložení škodlivého skriptu do stránky. Django automaticky escapuje výstup šablon.
  • CSRF (Cross-Site Request Forgery) – podvržení požadavku. Django používá CSRF token.
  • SQL Injection – vložení škodlivého SQL kódu. ORM chrání automaticky.
  • Brute force – hádání hesla opakovanými pokusy.

14.5 CMS, trendy a nasazení webových aplikací

CMS (Content Management System)

Redakční systémy pro správu obsahu webu. Příklady: WordPress, Django CMS.

Moderní trendy

  • SPA (Single Page Application) – nenačítá celou stránku při přechodu; React, Angular, Vue.
  • Headless CMS – backend zcela oddělen od frontendu; komunikace přes API.
  • Serverless – provoz v cloudu bez nutnosti správy serveru.
  • PWA (Progressive Web App) – webovka s vlastnostmi mobilní aplikace (offline režim, notifikace).

Možnosti nasazení

  • Webhosting – sdílený prostor na serveru (Wedos, Active24).
  • VPS (Virtual Private Server) – virtuální server s plnou kontrolou.
  • Cloud / PaaS – platforma jako služba (Heroku, AWS Elastic Beanstalk).
  • Dedikovaný server – pronajatý fyzický hardware.
