Vývojářské nástroje a verzovací systémy

17.1 Fáze tvorby programu a vývojářské nástroje

  1. Návrh algoritmu a programové architektury – používají se tzv. UML (Unified Modeling Language), standardizovaný jazyk sloužící pro vizualizaci a správu různých aspektů SW systémů, používá diagramy. Příklady: draw.io, open-source varianta DIA (lze použít k tvorbě ER diagramů).
  2. Psaní a ladění kódu – hojně se využívají IDE, která poskytují programátorům celou sadu funkcí a pomůcek. Příklady: PyCharm, Visual Studio Code.
  3. Testování – nástroje jako xUnit (unit testy) nebo Postman (testování API).
  4. Správa verzí projektu – pro správu projektu, spolupráci a sledování změn se používají verzovací systémy, např. Git, který nabízí správu lokálních i vzdálených repositářů.
  5. Dokumentace – klíčová pro správnou údržbu. Např. Doxygen umožňuje tvorbu dokumentace z poznámek v kódu.

17.2 IDE, pomůcky pro editaci a refaktorování kódu

IDE (Integrated Development Environment) je integrované vývojářské prostředí poskytující programátorovi komplexní sadu nástrojů a funkcí pro vývoj SW. Kromě textového editoru zahrnuje také:

  • Debugger – umožňuje vývojáři chronologicky procházet a ladit kód.
  • Profiler – identifikuje slabá místa v kódu, optimalizuje výkon.
  • Integrace s verzovacím systémem – umožňuje pracovat s různými verzemi kódu, zapojit tým.
  • Sestavování a testování – sestavení aplikace z kódu a spuštění automatického testu.
  • Správce balíčků – snadná instalace, odinstalace a obecná správa balíčků a knihoven.

Užitečné pomůcky IDE

  • Syntax zvýraznění – zvyšuje čitelnost a zpřehledňuje kód (barevně odlišuje proměnné).
  • Automatické dokončování kódu – minimalizace chyb a urychlení práce.
  • Code snippets – krátké úryvky kódu, které lze vložit do projektu.
  • Code folding – skrytí pasáže kódu, zůstane jen hlavička; zvyšuje přehlednost u delších kódů.
  • Linter – upozorňuje na potenciální chyby; integrován do textového editoru.
  • Copilot – umělá inteligence, generuje kód na základě vývojářského vstupu; snaha zrychlit vývoj; dnes nabízí generaci kódů v jazycích Python, JavaScript, TypeScript a další.

Refaktorování kódu je estetická úprava kódu bez změnění jeho vnějšího chování; snaha zlepšit přehlednost a čitelnost. Postupuje se systematicky: změny jmen proměnných, rozdělení funkcí na co nejmenší části, zefektivnění algoritmu atd.

Zásady psaní kódu

  • DRY (Do not Repeat Yourself) – minimalizovat opakování; každá informace má jediné a jasně definované zdrojové místo v kódu.
  • KISS (Keep it Simple, Stupid) – oproštění se od zbytečně složitého kódu; složitá řešení jsou náchylná k chybám.
  • YAGNI (You Ain't Gonna Need It) – nepíšeme kód pro případ, který nemusí nastat.
  • SOLID – soubor zásad pro psaní objektově orientovaného kódu.

17.3 Nástroje pro ladění a testování

Debugger je nástroj používaný k hledání a ladění chyb v kódu; umožňuje programátorovi spustit kód a postupovat krok po kroku a zjišťovat, jaký efekt mají jednotlivé úseky na zbytek aplikace.

  • Zastavení programu v určitém bodě.
  • Krok po kroku – volitelná rychlost postupu.
  • Sledování hodnot proměnných – lze je sledovat a porovnávat v libovolné části programu.
  • Stack trace – sledování stavu zásobníku volání funkcí.
  • Profilování výkonu – délka trvání vykonání jednotlivých dílčích funkcí.

Testování kódu ověřuje správnost, minimalizuje negativní dopad a zvyšuje kvalitu výsledku.

TDD (Test Driven Development) – vývoj řízený testy; píše testy ještě před psaním samotného kódu; aplikace je tedy spolehlivější a efektivnější.

17.4 Správa závislostí, virtuální prostředí a kontejnery

Závislosti mohou být vnitřní (různé části jednoho programu) nebo vnější (program a externí knihovna). Správa závislostí je důležitou součástí moderního programování. Nástroje:

  • pip – Python; instalace balíčků; pip freeze; instalace pomocí přímého odkazu na Git.
  • npm – JS a Node.js; instalace z centrálního registru npm a externích zdrojů (Git).
  • yarn – alternativa k npm; vylepšená výkonnost, možnost paralelního stahování balíčků.

Virtuální prostředí odděluje závislosti projektu od systémových knihoven a nástrojů; projekt se stává nezávislý na konkrétním PC, snadno přenositelný. Python: python -m venv <název>, <název>\Scripts\activate.

Kontejnery jsou virtuální prostředí sloužící pro běh aplikací. Kontejner izoluje zdroje (OS, síť, souborové systémy) od hostitelského systému; jsou velmi flexibilní a snadno přenosné; používají se pro běh aplikací na cloudu nebo testování.

  • Docker – nejznámější a nejčastěji používaná platforma pro kontejnerizaci.
    • Docker image – základní stavební jednotka; statický balíček obsahující vše potřebné.
    • Docker compose – umožňuje definovat a spustit více kontejnerů v jednom souboru.

Výhody kontejnerizace

  • Izolace – závislostí a aplikací od hostitelského systému.
  • Konzistence – minimalizace chyb spojených s různými verzemi knihoven.
  • Přenosnost – nezávislé na OS, SW i HW.
  • Efektivita – sdílení zdrojů mezi aplikacemi.
  • Škálovatelnost, snadná správa.

17.5 Verzovací systém – funkce a využití

Jedná se o nástroj pro sledování změn v kódu a zajištění správného řízení verzí. Základní funkce:

  1. Správa verzí – ukládají různé verze souborů; programátor dokáže pracovat v paralelních větvích a následně je spojit.
  2. Sledování změn – sledují a ukládají změny jako revize; uživatel může sledovat vývoj kódu a rozdíly mezi verzemi.
  3. Kolaborace – práce více uživatelů spojena do jediného souboru.
  4. Zpětné zotavení – obnovení předchozí verze v případě chyby nebo ztráty.
  5. Větvení a slučování – vytváření větví pro paralelní vývoj, lze je slučovat; jeden vývojář řeší problém A, druhý problém B a následně je spojí.

Git je distribuovaný verzovací systém sloužící pro správu kódu a souborů. Ukládá celou historii projektu; lze se kdykoliv vrátit k předchozímu stavu. Používá snapshots, které ukládají změny od předchozí provedené změny. Má decentralizovanou architekturu – každý vývojář má lokální kopii projektu, může pracovat off-line.

  • GitHub – bezplatné hostování veřejných repositářů, nabízí také placené plány.
  • GitLab – větší možnost pro správu a konfiguraci než GitHub.
  • Bitbucket – hostuje Git repositáře v cloudu i na vlastních serverech.

Základní příkazy Git

  • git add – přidá změny v pracovním adresáři do indexu.
  • git commit – vytvoří nový commit ze změn uložených v indexu.
  • git push – nahraje lokální změny do repositáře.
  • git pull – stáhne všechny změny z repositáře, sloučí je s lokálními soubory.
  • git clone – vytvoří kopii vzdáleného repositáře.
  • git status – zobrazí aktuální stav (změněné soubory, aktuální větev).
  • git diff – zobrazí rozdíly mezi aktuálním a posledním commitnutým stavem.
  • git log – seznam commitů v repositáři, lze filtrovat.
  • git remote – zobrazí seznam vzdálených repositářů a zkontroluje jejich nastavení.
  • git show – zobrazí autora, datum, čas, zprávu atd. daného commitu.

Větve v Git

Slouží k oddělení různých funkcionalit, oprav a vylepšení od hlavní linie (master větev). Umožňují na sobě nezávislé větve, které lze spojit. Podporují práci v týmu a experimentování bez rizika ovlivnění hlavní větve.

  • git branch – seznam všech větví repositáře.
  • git checkout – přepínání mezi větvemi.
  • git merge – zanoření větve do té hlavní.
