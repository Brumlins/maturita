df, mount — souborové systémy

🐧 Linux 💾 disk management

df — volné místo na discích

Zjištění volného místa 
# Přehled všech připojených disků (human-readable):
$ df -h

# Pouze reálné souborové systémy (bez tmpfs):
$ df -hT --exclude-type=tmpfs

# Konkrétní adresář:
$ df -h /var/www

# Inode statistiky:
$ df -i
SloupecPopis
Filesystemnázev zařízení nebo souborového systému
Sizecelková velikost
Usedvyužité místo
Availvolné místo
Use%procento využití
Mounted onpřípojný bod

du — velikost adresářů a souborů

Velikost adresářů 
# Celková velikost adresáře:
$ du -sh /var/log

# Největší podadresáře:
$ du -h --max-depth=1 /var | sort -hr | head -20

# Top 10 největších souborů v /var/log:
$ du -ah /var/log | sort -hr | head -10

mount / umount

mount připojí souborový systém nebo zařízení do adresářového stromu.
Připojení a odpojení 
# Zobrazit všechna připojená fs:
$ mount
# nebo:
$ cat /proc/mounts

# Připojit oddíl:
$ mount /dev/sdb1 /mnt/data

# Připojit ISO obraz:
$ mount -o loop obraz.iso /mnt/iso

# Odpojit:
$ umount /mnt/data
# Násilně (pokud je busy):
$ umount -l /mnt/data
Typ fsPoužití
ext4Linux standard (lokální disk)
ntfsWindows disky (ntfs-3g)
vfatFAT32 (USB flash disky)
nfssíťové sdílení NFS
cifsSamba / Windows share
tmpfsRAM disk (dočasný)

/etc/fstab — automatické připojení

Soubor /etc/fstab definuje souborové systémy připojované při startu systému.
Formát záznamu v /etc/fstab 
# Zařízení        Přípojný bod  Typ    Možnosti        dump  pass
/dev/sdb1         /mnt/data     ext4  defaults        0 2
UUID=abc123       /mnt/disk     ext4  defaults        0 2

# UUID disku zjistíš příkazem:
$ blkid

# Otestovat fstab bez restartu:
$ mount -a
// pass hodnoty 0 = nekontrolovat fsck při startu. 1 = root fs. 2 = ostatní (zkontrolovat po rootu).

lsblk / fdisk — přehled disků

Přehled blokovacích zařízení 
# Přehledný strom disků a oddílů:
$ lsblk

# S velikostí a přípojnými body:
$ lsblk -f

# Výpis partition table:
$ sudo fdisk -l
# nebo pro konkrétní disk:
$ sudo fdisk -l /dev/sda
