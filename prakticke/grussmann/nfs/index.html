NFS server

🐧 Linux 📂 Network File System

Co je NFS

NFS (Network File System) je protokol pro sdílení souborů přes síť na Linuxu/Unixu. Vzdálený adresář se připojí jako lokální — aplikace nepoznají rozdíl. Oproti Sambě je NFS jednodušší, ale funguje pouze mezi Linux/Unix systémy.

Server — instalace a příprava adresáře

Instalace NFS serveru 
$ apt-get install nfs-kernel-server

# Vytvoření sdíleného adresáře:
$ mkdir /data
$ chmod 777 /data

# Nebo s konkrétním vlastníkem:
$ chown nobody:nogroup /data

/etc/exports — konfigurace sdílení

Soubor /etc/exports definuje, které adresáře se sdílejí a s jakými právy.
/etc/exports 
# Formát: adresář  klient(možnosti)

# Přístup všem (*), zápis povolen, bez omezení roota:
/data *(rw,no_root_squash,no_subtree_check,sync)

# Přístup konkrétní IP:
/data 192.168.0.10(rw,sync,no_subtree_check)

# Přístup celé síti:
/data 192.168.0.0/24(rw,sync,no_subtree_check)
MožnostPopis
rwčtení i zápis (ro = pouze čtení)
synczápis potvrzen až po fyzickém zapsání na disk (bezpečnější)
no_root_squashroot klient má root práva na serveru (jinak je mapován na nobody)
no_subtree_checknezkoumá podstrom — rychlejší, doporučeno
Aplikace změn + restart serveru 
# Načíst změny v /etc/exports bez restartu:
$ exportfs -ra

# Nebo restart NFS:
$ systemctl restart nfs-kernel-server
# nebo starší:
$ /etc/init.d/nfs-kernel-server restart

# Ověření aktivních sdílení:
$ exportfs -v

Klient — připojení NFS sdílení

Instalace a ruční připojení 
$ apt-get install nfs-common

# Vytvoření přípojného bodu:
$ mkdir /media/nfs

# Připojení:
$ mount -t nfs 192.168.0.10:/data /media/nfs

# Ověření:
$ df -h
$ ls /media/nfs
Zobrazení dostupných sdílení na serveru 
$ showmount -e 192.168.0.10

Trvalé připojení přes /etc/fstab

Pro automatické připojení NFS sdílení při startu systému přidej záznam do /etc/fstab.
/etc/fstab 
# Přidat na konec /etc/fstab:
192.168.0.10:/data   /media/nfs   nfs   defaults,_netdev   0 0

# Testovat bez restartu (připojí vše z fstab):
$ mount -a
// _netdev Volba _netdev říká systému, aby počkal na spuštění sítě před připojením — nutné pro síťová úložiště.
