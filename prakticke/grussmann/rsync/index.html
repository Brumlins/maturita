rsync

🐧 Linux 📁 záloha & synchronizace

Co je rsync

rsync (Remote Sync) je CLI nástroj pro kopírování, zálohování a synchronizaci adresářů na Unix strojích. Klíčová výhoda: přenáší pouze změněné části souborů, nikoli celé soubory — proto je rychlejší než scp.

Podporuje: hardové i softové linky, přenos práv, skupin a vlastníků. Může využívat kompresi, takže šetří bandwidth.

Nejdůležitější parametry

ParametrFunkce
-vverbose — výpis informací o přenosu
-rrekurzivní kopírování (nezachová práva a časy)
-aarchivní mód = -rlptgoD (zachová práva, časy, skupiny, symlinky)
-zkomprese dat při přenosu — šetří bandwidth
-hhuman-readable — zobrazuje rychlost a velikost čitelně
-e sshpoužít SSH jako přenosový protokol (šifrování)
--progresszobrazovat průběh přenosu (%, rychlost, zbývající čas)
--deletesmaže v cíli soubory, které nejsou v zdroji
--dry-runsimulace — nic nezmění, jen ukáže co by se stalo
--bwlimit=Nomezení I/O bandwidth na N KB/s
--max-size=Npřenést pouze soubory do velikosti N (např. 200k, 1M)
--include/--excludefiltrování souborů podle vzoru
-Wpřenos celého souboru (bez delta algoritmu)
--remove-source-filespo přenosu smaže zdrojové soubory
// TIP Nejčastější kombinace pro bezpečnou zálohu: rsync -avzh --progress zdroj/ cil/

Instalace

Debian/Ubuntu 
$ apt-get install rsync

# Red Hat/CentOS:
$ yum install rsync

Lokální kopírování

Kopírování jednoho souboru 
# rsync options source destination
$ rsync -zvh backup.tar /tmp/backups/
# rsync vytvoří cílový adresář automaticky, pokud neexistuje
Kopírování celého adresáře (archivní mód) 
$ rsync -avzh /root/data/ /tmp/backups/
# -a zachová práva, časy, vlastníky
# Pozor: rsync /src/ dest/ zkopíruje obsah, rsync /src dest/ zkopíruje adresář

Přenos na vzdálený server

Z lokálního na vzdálený 
$ rsync -avz data/ root@192.168.0.101:/home/backup/
Ze vzdáleného na lokální 
$ rsync -avzh root@192.168.0.100:/home/data/ /tmp/local-backup/

Rsync přes SSH (šifrovaný přenos)

Přenos souboru přes SSH 
$ rsync -avzhe ssh root@192.168.0.100:/root/install.log /tmp/
# -e ssh = použít SSH jako přenosový protokol
Přenos s průběhem 
$ rsync -avzhe ssh --progress /home/data/ root@192.168.0.100:/backup/
# Zobrazuje: jméno souboru, %, rychlost, zbývající čas

Pokročilé možnosti

Include / Exclude filtrování 
# Přenést pouze soubory začínající na R, zbytek ignorovat
$ rsync -avze ssh --include 'R*' --exclude '*'
      root@192.168.0.101:/var/lib/rpm/ /root/rpm/
--delete (mirror záloha) 
# Smaže v cíli soubory, které nejsou ve zdroji — přesné zrcadlo
$ rsync -avz --delete root@192.168.0.100:/var/lib/rpm/ .
--dry-run (testovací běh) 
# Nic nezmění, jen ukáže co by rsync udělal
$ rsync --dry-run -avzh /etc/ /tmp/test-backup/
Omezení velikosti souborů a bandwidth 
# Přenést jen soubory menší než 200 kB
$ rsync -avzhe ssh --max-size='200k' /var/log/ root@server:/backup/

# Omezit bandwidth na 100 KB/s
$ rsync --bwlimit=100 -avzhe ssh /data/ root@server:/backup/
Automatické smazání zdrojového souboru po přenosu 
# Přenese soubor a pak ho smaže ze zdroje (move místo copy)
$ rsync --remove-source-files -zvh backup.tar /tmp/backups/
// PŘENOS CELÉHO SOUBORU Parametr -W (--whole-file) přenese soubor celý místo jen delta změn. Užitečné při pomalé síti, kde je delta algoritmus neefektivní. rsync -zvhW backup.tar /tmp/backups/backup.tar
