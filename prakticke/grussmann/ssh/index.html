SSH

🐧 Linux 🔐 bezpečný vzdálený přístup

1 — Instalace SSH serveru

Debian/Ubuntu 
$ apt-get install openssh-server

# Ověření stavu:
$ systemctl status ssh
$ systemctl enable ssh   # spustit při startu

2 — Generování SSH klíče

SSH klíče fungují na principu asymetrické kryptografie: privátní klíč zůstane u tebe, veřejný klíč nahraješ na server. Přihlášení pak funguje bez hesla.
Generování klíče (RSA 4096) 
$ ssh-keygen -t rsa -b 4096
# Zeptá se na umístění (výchozí ~/.ssh/id_rsa) a passphrase
# Passphrase = heslo pro odemčení klíče (doporučeno, ale není nutné)

# Výsledek:
#   ~/.ssh/id_rsa       ← privátní klíč (NIKDY neposílat!)
#   ~/.ssh/id_rsa.pub   ← veřejný klíč (jde nahrát na servery)
// TIP Různé algoritmy: -t rsa (nejběžnější), -t ed25519 (moderní, kratší, bezpečnější), -t dsa (starší, nedoporučeno).

3 — Kopírování veřejného klíče na server

Metoda 1 — ssh-copy-id (nejjednodušší) 
$ ssh-copy-id uzivatel@192.168.0.10
# Automaticky přidá ~/.ssh/id_rsa.pub do ~/.ssh/authorized_keys na serveru
Metoda 2 — přes scp ručně 
$ scp ~/.ssh/id_rsa.pub uzivatel@192.168.0.10:~/.ssh/authorized_keys

# Nebo ručně na serveru:
server$ cat ~/id_rsa.pub >> ~/.ssh/authorized_keys
server$ chmod 600 ~/.ssh/authorized_keys
server$ chmod 700 ~/.ssh/

4 — Přihlášení přes SSH klíč

Přihlášení 
$ ssh uzivatel@192.168.0.10
# Automaticky použije ~/.ssh/id_rsa

# Specifikovat konkrétní klíč:
$ ssh -i ~/.ssh/id_rsa_server uzivatel@192.168.0.10

5 — Zákaz přihlášení heslem

Po nahrání SSH klíčů doporučeno zakázat přihlašování heslem — ochrana před brute-force útoky.
/etc/ssh/sshd_config 
# Upravit /etc/ssh/sshd_config:
PasswordAuthentication no
PermitRootLogin no          # doporučeno i toto
PubkeyAuthentication yes

# Restart SSH:
$ systemctl restart ssh
# nebo na starších systémech:
$ /etc/init.d/ssh restart
// POZOR Než zakážeš hesla, ověř že se klíčem opravdu dokážeš přihlásit — jinak se uzamkneš ze serveru.

6 — Kopírování souborů přes SCP

SCP — Secure Copy 
# Nahrát soubor na server:
$ scp /lokalni/soubor.txt uzivatel@server:/vzdáleny/adresár/

# Stáhnout soubor ze serveru:
$ scp uzivatel@server:/vzdáleny/soubor.txt /lokalni/cil/

# Rekurzivní kopírování adresáře:
$ scp -r /lokalni/adresar/ uzivatel@server:/cil/

7 — Pokročilé: SSH tunel, SSHFS, socket

SSH tunel (port forwarding) 
# Local port forwarding: localhost:8080 → server:80
$ ssh -L 8080:localhost:80 uzivatel@server

# SOCKS proxy (socket server) na portu 1234:
$ ssh -D 1234 uzivatel@server
# Lze pak nastavit v Firefox jako SOCKS5 proxy na localhost:1234
SSHFS — připojení vzdáleného adresáře 
$ apt-get install sshfs

# Připojení:
$ sshfs -o idmap=user uzivatel@server:~/sdilena/ ~/mount-point/

# Odpojení:
$ fusermount -u ~/mount-point/
Spuštění příkazu přes SSH (bez interaktivního shellu) 
$ ssh uzivatel@server "ls -la /var/log/"
$ ssh uzivatel@server "df -h"

8 — Zabezpečení: iptables + fail2ban

iptables — omezení SSH přístupu na konkrétní IP 
# Flush (vymazat pravidla)
$ iptables -F

# Povolit SSH pouze z localhost (nebo konkrétní IP)
$ iptables -A INPUT -s 192.168.0.0/24 -p tcp --dport 22 -j ACCEPT

# Odmítnout SSH ze všech ostatních zdrojů
$ iptables -A INPUT -p tcp -s 0/0 --dport 22 -j REJECT
fail2ban — ochrana před brute-force 
$ apt-get install fail2ban
$ systemctl enable fail2ban
# Automaticky blokuje IP po N neúspěšných pokusech o přihlášení
# Konfigurace: /etc/fail2ban/jail.conf (nebo jail.local)
