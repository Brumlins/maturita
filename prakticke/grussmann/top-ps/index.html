top — live monitoring procesů

top zobrazuje živé informace o běžících procesech, CPU a paměti. Aktualizuje se každé 3 sekundy.

htop — vylepšený top

Instalace a spuštění htop 
$ apt-get install htop
$ htop

# F2 = nastavení, F6 = řazení, F9 = kill, F10 = quit
// htop vs top htop má barevné TUI, přehledné CPU/RAM grafy, myš, stromové zobrazení procesů, jednodušší ovládání. Pro denní práci preferovaný.

ps — snapshot procesů

ps vypíše aktuální procesy jako statický snapshot (narozdíl od top). Hodí se pro skripty.
Časté použití ps 
# Všechny procesy v rozšířeném formátu:
$ ps aux

# Procesy jako strom (parent/child):
$ ps axjf
# nebo:
$ pstree

# Najít konkrétní proces (kombinace s grep):
$ ps aux | grep apache2

# Zjistit PID procesu podle jména:
$ pgrep apache2

kill / killall — ukončení procesů

Zabití procesů 
# Ukončit proces podle PID (SIGTERM = gentle stop):
$ kill 1234

# Násilně ukončit (SIGKILL = okamžité zabití):
$ kill -9 1234
# nebo:
$ kill -SIGKILL 1234

# Zabít všechny procesy daného jména:
$ killall apache2
# nebo:
$ pkill apache2
SignálČísloPopis
SIGTERM15žádost o ukončení (defaultní, čistě)
SIGKILL9okamžité ukončení (nejde ignorovat)
SIGHUP1reload konfigurace (pro daemony)

nice / renice — priorita procesů

Priorita procesu (nice value) určuje, jak moc CPU dostane. Rozsah: -20 (nejvyšší) až +19 (nejnižší). Normální procesy: 0. Vyšší nice = nižší priorita.
Nastavení priority 
# Spustit příkaz s nízkou prioritou (+19):
$ nice -n 19 tar cvfz backup.tar.gz /home

# Změnit prioritu běžícího procesu:
$ renice +10 -p 1234

# Zvýšit prioritu (vyžaduje root):
$ sudo renice -5 -p 1234
