Vytvoření tabulky sponzor
Vytvoř tabulku sponzor pro evidenci sponzorů tělovýchovné jednoty. Struktura vychází z tohoto vzorového záznamu:

Vytvoř tabulku přes PHPMyAdmin klikačkou.
// POSTUP V PHPMyAdmin (krok za krokem)
  1. V levém panelu vyber databázi tj, klikni na Nová tabulka.
  2. Zadej název: sponzor, počet sloupců: 6 → klikni Provést.
  3. Vyplň každý sloupec (zleva: Název / Typ / Délka / Výchozí / Null / Index / A_I):
NázevTypDélka / HodnotyVýchozíNull ✔IndexA_I
idsponzorINT———PRIMARY✔
nazevVARCHAR100————
prispevekDECIMAL10,2NULL✔——
obecVARCHAR50NULL✔——
smlouva_odDATE——✔——
smlouva_doDATE——✔——
  1. U idsponzor – zaškrtni A_I → PHPMyAdmin navrhne PRIMARY index, potvrď.
  2. Po vyplnění všech sloupců klikni dole na Uložit.
SQL 
CREATE TABLE sponzor (
  idsponzor  INT           NOT NULL AUTO_INCREMENT,
  nazev      VARCHAR(100)  NOT NULL,
  prispevek  DECIMAL(10,2) DEFAULT NULL,
  obec       VARCHAR(50)   DEFAULT NULL,
  smlouva_od DATE          DEFAULT NULL,
  smlouva_do DATE          DEFAULT NULL,
  PRIMARY KEY (idsponzor)
) ENGINE=InnoDB
  DEFAULT CHARSET=utf8
  COLLATE=utf8_unicode_ci;
// KLÍČOVÉ VĚCI
  • DECIMAL(10,2) – max. 10 číslic, 2 des. místa → max. 99 999 999.99 Kč
  • Ostatní sloupce jsou nepovinné (DEFAULT NULL) – smlouva nemusí být ihned
  • AUTO_INCREMENT – ID sponzora se samo zvyšuje, nemusíš ho zadávat

Sportovní akce s vstupným v Opavě
Vypiš všechny sportovní akce konané v Opavě, za které bylo vybíráno vstupné. Zobraz název akce, datum konání, pořádající oddíl a výši vstupného. Seřaď podle data vzestupně.
SQL 
SELECT
  a.nazev                                      AS `Název akce`,
  DATE_FORMAT(a.datum_konani, '%e. %c. %Y')    AS `Datum konání`,
  o.nazev                                      AS `Pořádající oddíl`,
  CONCAT(a.vstupne, ' Kč')                    AS `Vstupné`

FROM akce a
JOIN oddily o ON o.zkratka = a.poradajici_oddil

WHERE a.typ    = 'sportovní'
  AND a.misto  = 'Opava'
  AND a.vstupne > 0

ORDER BY a.datum_konani ASC;
// JAK TO FUNGUJE
  • JOIN oddily – abychom mohli zobrazit celý název oddílu místo zkratky (FK, AK...)
  • typ = 'sportovní' – ENUM sloupec: sportovní/společenská/kulturní/jiná
  • vstupne > 0 – filtrujeme akce s kladným vstupným (akce bez vstupného mají 0.00)
  • CONCAT(vstupne, ' Kč') – přidá jednotku k číslu

Předsedové oddílů
Vypiš jména předsedů všech oddílů včetně názvu oddílu a data přijetí do TJ. Seřaď dle názvu oddílu abecedně.
SQL 
SELECT
  CONCAT(l.jmeno, ' ', l.prijmeni) AS `Jméno`,
  o.nazev                          AS `Oddíl`,
  DATE_FORMAT(c.datum_prijeti, '%e. %c. %Y') AS `Datum přijetí`

FROM clenove c
JOIN lide   l  ON l.rodne_cislo = c.lide_rodne_cislo
JOIN oddily o  ON o.zkratka     = c.oddil_zkratka

WHERE c.funkce = 'předseda'

ORDER BY o.nazev ASC;
// JAK TO FUNGUJE
  • Osobní data jsou v lide, propojení přes rodné číslo (lide_rodne_cislo)
  • JOIN oddily – přidá název oddílu místo zkratky FK, BK…
  • WHERE funkce = 'předseda' – funkce je FK do tabulky funkce
  • Každý oddíl má jednoho předsedu → výsledek má 5 řádků

Počet členů v každém oddílu – GROUP BY
Vypiš přehled oddílů s celkovým počtem jejich členů. Seřaď primárně podle počtu členů (sestupně), sekundárně podle názvu oddílu (abecedně).
SQL 
SELECT
  o.nazev                AS `Oddíl`,
  o.odvetvi              AS `Odvětví`,
  COUNT(c.cislo_prukazu) AS `Počet členů`

FROM oddily o
LEFT JOIN clenove c ON c.oddil_zkratka = o.zkratka

GROUP BY o.zkratka

ORDER BY `Počet členů` DESC,
         o.nazev         ASC;
// KLÍČOVÉ KONCEPTY
  • LEFT JOIN – zahrne i oddíl bez členů (kdyby takový byl); COUNT pak vrátí 0
  • GROUP BY o.zkratka – seskupujeme podle zkratky oddílu (unikátní PK), ne názvem
  • COUNT(c.cislo_prukazu) – počítá záznamy v tabulce clenove pro daný oddíl
  • Při shodě počtu (Boxerský i Fotbalový mají 6) rozhoduje abecední řazení názvů

Akce s více než 200 účastníky
Vypiš všechny akce, jejichž se zúčastnilo více než 200 lidí. Zobraz název akce, datum, typ, pořádající oddíl a počet účastníků. Seřaď dle počtu účastníků sestupně.
SQL 
SELECT
  a.nazev                                      AS `Název akce`,
  DATE_FORMAT(a.datum_konani, '%e. %c. %Y')    AS `Datum`,
  a.typ                                        AS `Typ`,
  o.nazev                                      AS `Pořádající oddíl`,
  a.pocet_ucastniku                            AS `Počet účastníků`

FROM akce a
JOIN oddily o ON o.zkratka = a.poradajici_oddil

WHERE a.pocet_ucastniku > 200

ORDER BY a.pocet_ucastniku DESC;
// JAK TO FUNGUJE
  • Jednoduchý SELECT s JOIN – pro každou akci s >200 účastníky zobrazíme i celý název oddílu
  • pocet_ucastniku > 200 – podmínka filtruje přímo v tabulce akce, není třeba GROUP BY/HAVING
  • Výsledek zahrnuje různé typy: kulturní, sportovní i společenská

h)
Smazání tabulky sponzor a celé databáze
Nejdřív odstraň tabulku sponzor, pak úplně smaž celou databázi tj.
SQL 
-- odstraní tabulku sponzor
DROP TABLE sponzor;

-- odstraní celou databázi se všemi tabulkami
DROP DATABASE tj;
// POZOR
  • DROP TABLE smaže tabulku včetně všech dat, nelze vrátit
  • DROP DATABASE smaže celou databázi se všemi tabulkami a daty
  • Pořadí záleží – nejdřív DROP TABLE, pak DROP DATABASE
