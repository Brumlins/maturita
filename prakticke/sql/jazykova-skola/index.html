  1. ISBN je VARCHAR (ne INT) – obsahuje pomlčky, primární klíč nastavíš výběrem PRIMARY v Index.
  2. U Uroven – vyber ENUM, do Délka/Hodnoty zadej: 'začátečník','mírně pokročilý','pokročilý'
  3. Po vyplnění všech sloupců klikni dole na Uložit.
SQL 
CREATE TABLE ucebnice (
  ISBN       VARCHAR(20)   NOT NULL,
  Titul      VARCHAR(100)  NOT NULL,
  Autor      VARCHAR(50)   DEFAULT NULL,
  Rok_vydani INT(4)        DEFAULT NULL,
  Uroven     ENUM('začátečník','mírně pokročilý','pokročilý') DEFAULT NULL,
  Popis      TEXT          DEFAULT NULL,
  Cena       DECIMAL(6,2)  DEFAULT NULL,
  PRIMARY KEY (ISBN)
) ENGINE=InnoDB
  DEFAULT CHARSET=utf8
  COLLATE=utf8_czech_ci;
// KLÍČOVÉ VĚCI
  • ISBN VARCHAR(20) – ISBN není číslo, obsahuje pomlčky → musí být řetězec
  • DECIMAL(6,2) – max. 6 číslic, 2 za desetinnou čárkou → max. 9999.99
  • ENUM('začátečník','mírně pokročilý','pokročilý') – jen povolené úrovně
  • Titul NOT NULL – zadání říká "povinně zadávaná hodnota", ostatní jsou nullable

Studenti kurzu „Angličtina – nic jednoduššího" z Brna nebo Ostravy
Vypiš všechny studenty přihlášené do kurzu Angličtina – nic jednoduššího, kteří pocházejí z Brna nebo Ostravy. Seřaď primárně podle bydliště (vzestupně), sekundárně podle data narození (od nejmladších).
SQL 
SELECT
  u.jmeno                                   AS `Jméno`,
  u.prijmeni                                AS `Příjmení`,
  u.bydliste                                AS `Bydliště`,
  DATE_FORMAT(u.narozeni, '%e. %c. %Y')    AS `Datum narození`

FROM studenti_v_kurzech svk
JOIN uzivatel u ON u.nick    = svk.uzivatel_nick
JOIN kurz k     ON k.idkurz  = svk.idkurz

WHERE k.nazev    = 'Angličtina - nic jednoduššího'
  AND u.bydliste IN ('Brno', 'Ostrava')

ORDER BY u.bydliste ASC,
         u.narozeni DESC;
// JAK TO FUNGUJE
  • JOIN uzivatel – z tabulky studenti_v_kurzech máme jen nick, jméno/bydliště jsou v uzivatel
  • JOIN kurz – filtrujeme podle názvu kurzu, ne podle ID
  • narozeni DESC – od nejmladšího = nejnovější datum nahoře

Kurzy angličtiny a němčiny pro začátečníky + počet studentů
Vypiš všechny kurzy angličtiny a němčiny pro začátečníky a k nim počet přihlášených studentů. Seřaď podle počtu studentů (sestupně).
SQL 
SELECT
  k.nazev                   AS `Název kurzu`,
  j.oznaceni               AS `Jazyk`,
  k.datum_zahajeni          AS `Zahájení`,
  COUNT(svk.uzivatel_nick) AS `Počet studentů`

FROM kurz k
JOIN jazyk j                  ON j.zkratka = k.jazyk
LEFT JOIN studenti_v_kurzech svk ON svk.idkurz = k.idkurz

WHERE k.jazyk IN ('ANJ', 'NEJ')
  AND k.uroven = 'začátečníci'

GROUP BY k.idkurz

ORDER BY `Počet studentů` DESC;
// KLÍČOVÉ KONCEPTY
  • LEFT JOIN studenti_v_kurzech – kurz se zobrazí i kdyby neměl žádného studenta
  • COUNT(svk.uzivatel_nick) – počet studentů na kurz; NULL hodnoty se nepočítají
  • GROUP BY k.idkurz – seskupujeme po kurzech, pak COUNT počítá za každý kurz zvlášť

Kurzy absolvované Miladou Frejovou
Vypiš všechny kurzy, které Milada Frejová absolvovala (datum ukončení není NULL). Seřaď vzestupně podle data ukončení kurzu.
SQL 
SELECT
  k.nazev          AS `Název kurzu`,
  j.oznaceni       AS `Jazyk`,
  k.uroven         AS `Úroveň`,
  k.datum_ukonceni AS `Rok ukončení kurzu`,
  svk.hodnoceni   AS `Hodnocení`

FROM studenti_v_kurzech svk
JOIN kurz     k ON k.idkurz  = svk.idkurz
JOIN jazyk    j ON j.zkratka = k.jazyk
JOIN uzivatel u ON u.nick    = svk.uzivatel_nick

WHERE u.jmeno    = 'Milada'
  AND u.prijmeni = 'Frejová'
  AND k.datum_ukonceni IS NOT NULL

ORDER BY k.datum_ukonceni ASC;
// JAK TO FUNGUJE
  • datum_ukonceni IS NOT NULL – kurzy s datem ukončení = absolvované kurzy
  • JOIN řetěz: studenti_v_kurzech → kurz → jazyk + uzivatel – 4 tabulky
  • Filtrujeme jménem a příjmením – nick „frejova" by šel použít taky, je to totéž

Vybavení a vytížení učeben – GROUP BY
Vypiš přehled učeben: jejich vybavení a počet kurzů, které se v nich konaly. Seřaď podle počtu kurzů (sestupně).
SQL 
SELECT
  u.oznaceni        AS `Označení`,
  u.pocet_mist      AS `Počet míst`,
  u.vybaveni        AS `Vybavení`,
  COUNT(k.idkurz)   AS `Počet kurzů`

FROM ucebna u
LEFT JOIN kurz k ON k.ucebna_oznaceni = u.oznaceni

GROUP BY u.oznaceni

ORDER BY `Počet kurzů` DESC;
// JAK TO FUNGUJE
  • LEFT JOIN kurz – učebna se zobrazí i kdyby neměla žádný kurz
  • GROUP BY u.oznaceni – seskupíme podle označení učebny, COUNT pak počítá kurzy za každou
  • COUNT(k.idkurz) – NULL hodnoty se nepočítají → učebna bez kurzů dostane 0

Snížení počtu lekcí o 2 u neukončených kurzů pro začátečníky
Sniž počet lekcí o 2 u všech kurzů, které dosud nebyly ukončeny (datum ukončení je NULL) a jsou určeny pro začátečníky.
SQL 
UPDATE kurz
SET pocet_lekci = pocet_lekci - 2
WHERE datum_ukonceni IS NULL
  AND uroven = 'začátečníci';
// CO SE ZMĚNÍ
  • Angličtina snadno a rychle: 10 → 8 lekcí
  • Ruština pro dobrodruhy: 10 → 8 lekcí
  • Angličtina – nic jednoduššího: 15 → 13 lekcí
  • Španělsky s prstem v nose: nezměněno – má uroven = 'pokročilí'

Smazání uživatelek narozených po roce 1980 bez „o" v bydlišti
Smaž z databáze všechny uživatelky narozené po roce 1980, které nepocházejí z místa, jehož název obsahuje písmeno o.
// PROČ NESTAČÍ JEDEN DELETE
  • Tabulky student a lektor odkazují na uzivatel přes FK s ON DELETE NO ACTION
  • Nejdřív musíme smazat záznamy v studenti_v_kurzech, pak v student
  • Pro lektorku lopezovou navíc: smazat z jazyk_lektoru a ošetřit kurzy (kurz na ni odkazuje)
  • Teprve po vyčištění vázaných tabulek lze smazat z uzivatel
SQL 
-- samotný dotaz na mazání uživatelek
DELETE FROM uzivatel
WHERE pohlavi  = 'žena'
  AND YEAR(narozeni) > 1980
  AND bydliste NOT LIKE '%o%';
// CO SE SMAŽE a PROČ
  • lopez (1988, Madrid) – Madrid neobsahuje 'o' → smazána
  • pilna (1992, Praha) – Praha neobsahuje 'o' → smazána
  • grey (1993, London) – London obsahuje 'o' → ponechána
  • seidlova, soudkova, volna – Ostrava/Olomouc obsahují 'o' → ponechány
  • NOT LIKE '%o%' – v MySQL case-insensitive, takže 'O' v Ostravě také odpovídá
