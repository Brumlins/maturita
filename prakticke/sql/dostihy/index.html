a)
Import databáze + oprava kódování
1 bod  ·  PHPMyAdmin
Vytvoř v MySQL databázi s názvem dostihy. Importuj soubor dostihy.sql. Ověř správnost kódování českých znaků – pokud jsou rozbité, oprav.
SQL 
-- 1. Vytvoř databázi
CREATE DATABASE dostihy
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_czech_ci;

-- 2. Vyber ji
USE dostihy;

-- 3. Potom importuj soubor přes PHPMyAdmin:
--    Databáze → Import → vybrat dostihy.sql → Kódování: utf8mb4
// PROČ TO TAK FUNGUJE
  • CHARACTER SET utf8mb4 – plná podpora UTF-8, zvládne i emoji a české znaky
  • COLLATE utf8mb4_czech_ci – řazení podle českých pravidel (ch za h, ž za z apod.), ci = case insensitive
  • Při importu v PHPMyAdmin musíš v nabídce kódování vybrat utf8mb4, jinak se háčky a čárky rozbijí
b)
Vytvoření tabulky trener
4 body
Vytvoř tabulku trener s atributy: idtrener (PK), titul (ENUM), jmeno, prijmeni (VARCHAR 50), narozeni (DATE), pocet_koni (výchozí 10), uspesnost (max 100.00), clanek (TEXT).
SQL 
-- Tabulku trener vytváříme přes PHPMyAdmin GUI,
-- jen CHECK constraint musíme zadat ručně přes SQL:

CREATE TABLE trener (
  idtrener   INT(11)       NOT NULL AUTO_INCREMENT,
  titul      ENUM('Bc.', 'Dr.', 'Ing.', 'Mgr.') DEFAULT NULL,
  jmeno      VARCHAR(50)  NOT NULL,
  prijmeni   VARCHAR(50)  NOT NULL,
  narozeni   DATE         DEFAULT NULL,
  pocet_koni DOUBLE       DEFAULT 10,
  uspesnost  DECIMAL(5,2)  DEFAULT NULL
               CHECK (uspesnost <= 100.00),
  clanek     TEXT         DEFAULT NULL,
  PRIMARY KEY (idtrener)
) ENGINE=InnoDB
  DEFAULT CHARSET=utf8
  COLLATE=utf8_czech_ci;
// KLÍČOVÉ VĚCI
  • ENUM('Bc.','Dr.',...) – jen povolené hodnoty, nic jiného nelze vložit
  • DECIMAL(5,2) – číslo max. 5 číslic, z toho 2 za desetinnou čárkou → max. 999.99
  • CHECK (uspesnost <= 100.00) – databáze sama odmítne vyšší hodnotu
  • DEFAULT 10 u pocet_koni – pokud nezadáš hodnotu, automaticky se doplní 10
  • AUTO_INCREMENT – id se samo zvyšuje, nemusíš ho zadávat při vkládání

Koně-valaši ze stájí NAU a GEW s písmenem "o"
4 body
Vypiš všechny koně-valachy ze stájí Naughton R.J. a Gest.Westerberg, jejichž jméno obsahuje písmeno o. Seřaď primárně podle stáje (sestupně), sekundárně podle jména (vzestupně). Přidej věk koně.
SQL 
SELECT
  k.jmeno_kone   AS `Jméno koně`,
  k.pohlavi      AS `Pohlaví`,
  CONCAT(YEAR(NOW()) - YEAR(k.narozeni), ' let') AS `Věk`,
  s.nazev_staje  AS `Název stáje`

FROM kun k
LEFT JOIN staj s ON s.zkratka_staje = k.staj_zkratka_staje

WHERE (k.staj_zkratka_staje LIKE 'GEW'
    OR k.staj_zkratka_staje LIKE 'NAU')
  AND k.pohlavi LIKE 'valach'
  AND k.jmeno_kone LIKE '%o%'

ORDER BY s.nazev_staje DESC,
         k.jmeno_kone ASC;
// JAK TO FUNGUJE
  • LEFT JOIN staj – připojíme tabulku stájí podle zkratky, abychom mohli zobrazit celý název
  • LIKE '%o%' – % je wildcard (cokoliv), takže hledáme "o" kdekoliv ve jménu
  • CONCAT(YEAR(NOW()) - YEAR(narozeni), ' let') – spočítá věk jako rozdíl roků a přidá text "let"
  • ORDER BY ... DESC, ... ASC – první řazení sestupně, druhé vzestupně
  • Aliasy k a s – zkratky pro tabulky, píšeme k.jmeno_kone místo kun.jmeno_kone

Statistika stájí – GROUP BY
5 bodů
Přehled stájí a statistika účastí jejich koní v dostizích. Seřaď podle počtu koní v dostizích (sestupně), pak podle počtu vítězství (sestupně).
SQL 
SELECT
  s.nazev_staje                           AS `Název stáje`,
  COUNT(kd.kun_idkun)                    AS `Počet koní v dostizích`,
  ROUND(AVG(kd.umisteni), 1)             AS `Průměrné umístění`,
  SUM(IF(kd.umisteni = 1, 1, 0))         AS `Počet vítězství`

FROM staj s
LEFT JOIN kun k  ON k.staj_zkratka_staje = s.zkratka_staje
LEFT JOIN kun_v_dostihu kd ON kd.kun_idkun = k.idkun

GROUP BY s.nazev_staje

ORDER BY `Počet koní v dostizích` DESC,
         `Počet vítězství` DESC;
// KLÍČOVÉ KONCEPTY
  • GROUP BY nazev_staje – seskupí řádky podle stáje, pak agregační funkce počítají za každou skupinu
  • COUNT(kd.kun_idkun) – počet řádků v kun_v_dostihu pro danou stáj = počet startů
  • AVG(kd.umisteni) – průměrné umístění, ROUND(..., 1) zaokrouhlí na 1 des. místo
  • SUM(IF(umisteni = 1, 1, 0)) – pro každý řádek: pokud 1. místo → přičti 1, jinak 0. Výsledek = počet vítězství
  • Řetěz staj → kun → kun_v_dostihu – musíme projít přes 3 tabulky pro kompletní data

Vítězové dostihů v roce 2024
5 bodů
Vypiš vítěze (umisteni = 1) všech dostihů konaných v roce 2024. Seřaď podle data konání (vzestupně).
SQL 
SELECT
  d.nazev                                         AS `Dostih`,
  DATE_FORMAT(d.cas_konani, "%e. %c. %Y")        AS `Datum konání`,
  CONCAT(k.jmeno_kone, " (", k.pohlavi, ")")       AS `Kůň`,
  s.nazev_staje                                   AS `Stáj`,
  CONCAT(LEFT(j.jmeno, 1), ". ", j.prijmeni)      AS `Jezdec`

FROM staj s
LEFT JOIN kun k           ON s.zkratka_staje      = k.staj_zkratka_staje
LEFT JOIN kun_v_dostihu kd ON k.idkun             = kd.kun_idkun
LEFT JOIN jezdec j         ON j.idjezdec           = kd.jezdec_idjezdec
LEFT JOIN dostih d         ON d.iddostih           = kd.dostih_iddostih

WHERE YEAR(d.cas_konani) = "2024"
  AND kd.umisteni = 1

ORDER BY d.cas_konani ASC;
// CO JE ZAJÍMAVÉ
  • DATE_FORMAT(cas_konani, "%e. %c. %Y") – formátuje datum: %e = den bez nuly, %c = měsíc bez nuly, %Y = 4-místný rok
  • CONCAT(LEFT(jmeno, 1), ". ", prijmeni) – zkrátí jméno na iniciálu, výsledek např. "M. Kubík"
  • YEAR(cas_konani) = "2024" – filtr na rok, alternativa: cas_konani BETWEEN '2024-01-01' AND '2024-12-31'
  • JOIN řetěz: staj → kun → kun_v_dostihu → jezdec + dostih – propojujeme 5 tabulek přes cizí klíče

Jezdci s více než 1 dostih v roce 2024
5 bodů
Vypiš jezdce, kteří v 2024 absolvovali více než jeden dostih. Seřaď primárně podle nejlepšího umístění (vzestupně), sekundárně podle váhy (těžší má přednost).
SQL 
SELECT
  CONCAT(j.jmeno, ' ', j.prijmeni)  AS `Jezdec`,
  CONCAT(j.vaha, ' kg')              AS `Váha`,
  COUNT(kd.dostih_iddostih)         AS `Počet dostihů`,
  SUM(d.delka)                      AS `Odjeto metrů`,
  MIN(kd.umisteni)                  AS `Nejlepší umístění`

FROM jezdec j
LEFT JOIN kun_v_dostihu kd ON j.idjezdec      = kd.jezdec_idjezdec
LEFT JOIN dostih d         ON kd.dostih_iddostih = d.iddostih

WHERE YEAR(d.cas_konani) = 2024

GROUP BY j.idjezdec

HAVING `Počet dostihů` > 1

ORDER BY `Nejlepší umístění` ASC,
         j.vaha DESC;
// HAVING vs WHERE – DŮLEŽITÉ!
  • WHERE filtruje před seskupením – použij ho na sloupce tabulky (rok, jméno...)
  • HAVING filtruje po seskupení – použij ho na výsledky agregací (COUNT, SUM, MIN...)
  • MIN(kd.umisteni) – nejnižší číslo umístění = nejlepší výsledek (1. místo = 1)
  • SUM(d.delka) – celkový počet odjetých metrů přes všechny dostihy v roce
  • GROUP BY j.idjezdec – seskupení podle ID jezdce (ne jménem, protože jména se mohou opakovat)
