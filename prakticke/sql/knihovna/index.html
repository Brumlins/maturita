Knihovna 📚

a)
Import databáze
V PHPMyAdmin vytvoř novou databázi s názvem knihovna, kódování utf8_unicode_ci a klikni na Vytvořit. Pak jdi na Import, vyber soubor knihovna.sql a potvrď. Ověř kódování – pokud se diakritika zobrazuje špatně, v nastavení databáze změň kolaci na utf8_unicode_ci.
b)
Vytvoření tabulky rezervace
Vytvoř tabulku rezervace pro evidenci rezervací knih čtenáři. Struktura vychází z tohoto vzorového záznamu:
Název atributuDoména atributuVzorový záznam
idrezervaceprimární klíč, automatické číslo1
cislo_prukazucelé číslo o délce 5 číslic4
isbnznakový řetězec, max. 20 znaků978-80-7335-123-6
datum_rezervaceplatné datum, povinný údaj1. 5. 2010
platnost_doplatné datum15. 5. 2010
stavvýběr z voleb „aktivní", „splněná", „zrušená" (výchozí: aktivní)aktivní
Vytvoř tabulku přes PHPMyAdmin klikačkou.
// POSTUP V PHPMyAdmin (krok za krokem)
  1. V levém panelu vyber databázi knihovna, klikni na Nová tabulka.
  2. Zadej název: rezervace, počet sloupců: 6 → klikni Provést.
  3. Vyplň každý sloupec (formulář má zleva: Název / Typ / Délka–Hodnoty / Výchozí / Null / Index / A_I):
NázevTypDélka / HodnotyVýchozíNull ✔IndexA_I
idrezervaceINT———PRIMARY✔
cislo_prukazuINT5————
isbnVARCHAR20————
datum_rezervaceDATE—————
platnost_doDATE——✔——
stavENUM'aktivní','splněná','zrušená'as defined: aktivní———
  1. U idrezervace – zaškrtni A_I → PHPMyAdmin navrhne PRIMARY index, potvrď.
  2. U stav – vyber ENUM, do Délka/Hodnoty zadej: 'aktivní','splněná','zrušená'
  3. U stav – výchozí hodnotu nastav na as defined: aktivní
  4. Po vyplnění všech sloupců klikni dole na Uložit.
SQL 
CREATE TABLE rezervace (
  idrezervace     INT         NOT NULL AUTO_INCREMENT,
  cislo_prukazu   INT(5)      NOT NULL,
  isbn            VARCHAR(20) NOT NULL,
  datum_rezervace DATE        NOT NULL,
  platnost_do     DATE        DEFAULT NULL,
  stav            ENUM('aktivní', 'splněná', 'zrušená') NOT NULL DEFAULT 'aktivní',
  PRIMARY KEY (idrezervace)
) ENGINE=InnoDB
  DEFAULT CHARSET=utf8
  COLLATE=utf8_unicode_ci;
// KLÍČOVÉ VĚCI
  • ENUM('aktivní','splněná','zrušená') – jen povolené hodnoty, nic jiného nelze vložit
  • DEFAULT 'aktivní' – každá nová rezervace je automaticky aktivní, dokud nezměníme stav
  • datum_rezervace NOT NULL – povinný údaj, bez data se záznam odmítne vložit
  • platnost_do DEFAULT NULL – nepovinné, rezervace nemusí mít definovanou expiraci

d)
Čtenáři z Opavy
Vypiš všechny čtenáře s bydlištěm v Opavě. Zobraz číslo průkazu, příjmení, jméno, titul a datum narození. Seřaď abecedně podle příjmení.
SQL 
SELECT
  cislo_prukazu AS `Číslo průkazu`,
  prijmeni      AS `Příjmení`,
  jmeno         AS `Jméno`,
  titul         AS `Titul`,
  narozeni      AS `Datum narození`

FROM ctenar

WHERE obec = 'Opava'

ORDER BY prijmeni ASC;
// POZNÁMKY
  • Jednoduchý dotaz bez JOIN – všechna potřebná data jsou v tabulce ctenar
  • Sloupec titul je ENUM – pro čtenáře bez titulu se zobrazí NULL
  • narozeni je DATE – zobrazí se ve formátu YYYY-MM-DD; pro hezký výstup lze použít DATE_FORMAT(narozeni, '%e. %c. %Y')

Nevrácené výpůjčky – čtenář, titul, datum půjčení
Vypiš všechny výpůjčky, které čtenáři dosud nevrátili (datum vrácení není vyplněno). Zobraz jméno čtenáře, titul knihy a datum půjčení. Seřaď podle data půjčení vzestupně.
SQL 
SELECT
  CONCAT(c.jmeno, ' ', c.prijmeni)           AS `Čtenář`,
  k.titul                                    AS `Titul knihy`,
  DATE_FORMAT(v.datum_pujceni, '%e. %c. %Y') AS `Datum půjčení`

FROM vypujcka v
JOIN ctenar   c  ON c.cislo_prukazu = v.cislo_prukazu
JOIN exemplar e  ON e.signatura     = v.signatura
JOIN kniha    k  ON k.isbn          = e.isbn

WHERE v.datum_vraceni IS NULL

ORDER BY v.datum_pujceni ASC,
         c.prijmeni     ASC;
// JAK TO FUNGUJE
  • datum_vraceni IS NULL – hledáme řádky, kde datum vrácení nebylo vyplněno = kniha dosud leží u čtenáře
  • Řetěz joinů: vypujcka → exemplar → kniha – výpůjčka se váže na konkrétní fyzický exemplář, ten se pak spojí s katalogem knih
  • DATE_FORMAT(datum, '%e. %c. %Y') – hezčí formát data: 23. 2. 2010 místo 2010-02-23
  • Sekundární řazení dle příjmení zajistí deterministické pořadí u půjček se stejným datem

Statistika výpůjček podle čtenáře – GROUP BY
Vypiš ke každému čtenáři celkový počet výpůjček a součet zaplacených poplatků. Seřaď primárně podle počtu výpůjček (sestupně), sekundárně podle příjmení (abecedně).
SQL 
SELECT
  CONCAT(c.jmeno, ' ', c.prijmeni) AS `Čtenář`,
  COUNT(v.signatura)              AS `Počet výpůjček`,
  SUM(v.poplatek)                AS `Celkový poplatek (Kč)`

FROM ctenar c
LEFT JOIN vypujcka v ON v.cislo_prukazu = c.cislo_prukazu

GROUP BY c.cislo_prukazu

ORDER BY `Počet výpůjček` DESC,
         c.prijmeni        ASC;
// KLÍČOVÉ KONCEPTY
  • LEFT JOIN – zahrne i čtenáře bez výpůjček (v datech ale takoví nejsou); INNER JOIN by byl také funkční
  • GROUP BY c.cislo_prukazu – seskupení podle ID čtenáře (ne jménem – jména se mohou opakovat)
  • COUNT(v.signatura) – počet řádků ve výpůjčkách pro daného čtenáře
  • SUM(v.poplatek) – součet všech poplatků; sloupec je DECIMAL, výsledek bude také DECIMAL

Autoři a jejich knihy s vydavatelstvím
Vypiš přehled všech autorů a knih, které napsali, včetně vydavatelství. Seřaď primárně podle příjmení autora, sekundárně podle titulu knihy.
SQL 
SELECT
  CONCAT(a.jmeno, ' ', a.prijmeni) AS `Autor`,
  k.titul                          AS `Titul`,
  vyd.nazev                        AS `Vydavatelství`

FROM autor a
JOIN autori_knih    ak  ON ak.id_autora        = a.id_autora
JOIN kniha          k   ON k.isbn             = ak.kniha_isbn
JOIN vydavatelstvi  vyd ON vyd.id_vydavatelstvi = k.id_vydavatelstvi

ORDER BY a.prijmeni ASC,
         k.titul    ASC;
// JAK TO FUNGUJE
  • Tabulka autori_knih je vazební – řeší vztah M:N (jeden autor → více knih, jedna kniha → více autorů)
  • Procházíme řetězec: autor → autori_knih → kniha → vydavatelstvi
  • Autoři s více knihami (Tolstoj, Čapek, King, Vančura) se v výsledku zobrazí vícekrát – jeden řádek na knihu
  • Alias vyd pro vydavatelstvi – zabrání konfliktu s případnými jinými tabulkami se sloupcem nazev

h)
Smazání tabulky rezervace a celé databáze
Nejdřív odstraň tabulku rezervace, pak úplně smaž celou databázi knihovna.
SQL 
-- odstraní tabulku rezervace
DROP TABLE rezervace;

-- odstraní celou databázi se všemi tabulkami
DROP DATABASE knihovna;
// POZOR
  • DROP TABLE smaže tabulku včetně všech dat, nelze vrátit
  • DROP DATABASE smaže celou databázi se všemi tabulkami a daty
  • Pořadí záleží – nejdřív tabulka, pak DB. Opačně by DROP TABLE selhal, protože DB by neexistovala
