ISO/OSI model (Lekce 4)

OSI (Open Systems Interconnection) model rozděluje síťovou komunikaci do 7 vrstev. Každá vrstva má jasně definovanou funkci a komunikuje pouze se sousedními vrstvami.
#VrstvaFunkcePříklady protokolů
7Aplikačnírozhraní pro aplikaceHTTP, FTP, DNS, SMTP
6Prezentačníkódování, šifrování, kompreseSSL/TLS, JPEG, ASCII
5Relačníspráva relací (session)NetBIOS, RPC
4Transportníspolehlivý přenos, segmentaceTCP, UDP
3Síťoválogické adresování, směrováníIP, ICMP, OSPF
2Linkováfyzické adresování (MAC), přístup k médiuEthernet, WiFi, PPP
1Fyzickápřenos bitů po fyzickém médiuRJ-45, optika, koax

Podrobnosti k jednotlivým vrstvám, PDU (Protocol Data Unit) na každé vrstvě: data → segment → paket → rámec → bit

TCP/IP model

TCP/IP model (DoD model) má 4 vrstvy a je praktickým základem internetu. Mapuje se na OSI model, ale vrstvy jsou sloučeny.
Vrstva TCP/IPOdpovídá OSIProtokoly
Aplikační5+6+7HTTP, FTP, DNS, SSH, SMTP
Transportní4TCP (spolehlivý), UDP (rychlý)
Internetová3IP, ICMP, ARP, OSPF
Síťového rozhraní1+2Ethernet, WiFi
TCP vs UDP: TCP — spolehlivý, spojovaný, handshake (SYN→SYN-ACK→ACK), potvrzení, opakování. Použití: HTTP, SSH, FTP. UDP — nespolehlivý, nespojovaný, rychlý. Použití: DNS, streaming, VoIP, hry.

Three-way handshake detail, čísla portů, well-known ports (80, 443, 22, 21, 25, 53...)

MAC adresy a vztah k IP

MAC (Media Access Control) adresa je fyzická adresa síťového rozhraní. Je 48bitová, zapsaná jako 6 dvojic hex číslic. ARP (Address Resolution Protocol) překládá IP adresy na MAC adresy.
Zobrazení MAC a ARP tabulky 
# Zobrazení MAC adresy síťového rozhraní:
$ ip link show
$ ifconfig

# ARP tabulka (IP → MAC):
$ arp -n
$ ip neigh
Formát MAC adresy: AA:BB:CC:DD:EE:FF
První 3 bajty = OUI (výrobce). Poslední 3 bajty = unikátní ID zařízení.
Broadcast MAC: FF:FF:FF:FF:FF:FF

IP adresy a podsítě

IPv4 adresa je 32bitové číslo (4 oktety). Podsítě se definují maskou nebo CIDR prefixem.
TřídaRozsahPrivátní rozsah
A1.0.0.0 – 127.255.255.25510.0.0.0/8
B128.0.0.0 – 191.255.255.255172.16.0.0/12
C192.0.0.0 – 223.255.255.255192.168.0.0/16
CIDR zápis: 192.168.1.0/24 = síť 192.168.1.x, maska 255.255.255.0, 254 hostů
Speciální adresy: .0 = adresa sítě, .255 = broadcast, .1 obvykle brána
