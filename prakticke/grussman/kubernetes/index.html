Kubernetes

☸️ K8s 🐳 orchestrace kontejnerů

1 — Co je Kubernetes

Kubernetes (K8s) je open-source systém pro automatizaci nasazení, škálování a správy kontejnerových aplikací. Původně vyvinutý Google, nyní spravovaný CNCF.
// TIP Kubernetes není náhrada za Docker, ale platforma, která Docker (nebo jiný runtime) využívá pro běh kontejnerů.

2 — Hlavní komponenty Kubernetes

Architektura 
# Control Plane (hlavní uzel):
#   - API Server (kubectl komunikuje s ním)
#   - etcd (úložiště konfigurace)
#   - Scheduler (plánuje umístění podů)
#   - Controller Manager (hlídá stav clusteru)

# Worker Node (pracovní uzel):
#   - Kubelet (agent běžící na uzlu)
#   - Kube-proxy (síťová pravidla)
#   - Container Runtime (Docker, containerd...)

# Objekty:
#   - Pod (nejmenší jednotka, 1+ kontejnery)
#   - Service (zpřístupnění podů)
#   - Deployment (správa replik podů)

3 — Minikube - lokální Kubernetes

Instalace a spuštění 
# Instalace Minikube
$ curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
$ install minikube-linux-amd64 /usr/local/bin/minikube

# Instalace kubectl
$ curl -LO "https://dl.k8s.io/release/$(curl -L -s https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl"
$ install kubectl /usr/local/bin/kubectl

# Spuštění Minikube
$ minikube start

# Kontrola stavu
$ kubectl cluster-info
$ kubectl get nodes

4 — Základní objekty a příkazy

Příklady kubectl příkazů 
# Deployment nginx
$ kubectl create deployment nginx --image=nginx

# Seznam podů
$ kubectl get pods

# Seznam deploymentů
$ kubectl get deployments

# Škálování (počet replik)
$ kubectl scale deployment nginx --replicas=3

# Expose jako Service
$ kubectl expose deployment nginx --port=80 --type=NodePort

# Získání URL (v Minikube)
$ minikube service nginx --url
