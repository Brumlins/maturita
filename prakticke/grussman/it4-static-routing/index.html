Statický routing

🌐 Routing 🔧 manuální směrování

1 — Co je statický routing

Statický routing znamená ruční konfiguraci směrovacích tabulek na směrovačích. Správce ručně definuje, kam se mají posílat pakety pro konkrétní sítě.
// TIP Statický routing je jednoduchý, ale neškáluje se pro velké sítě. Vhodný pro malé sítě nebo specifické cesty.

2 — Konfigurace statického routingu (Cisco)

Cisco IOS příkazy 
# Příkaz ip route:
# ip route [cílová_síť] [maska] [další_hop | výstupní_rozhraní]

# Příklad 1: Směrování na konkrétní IP (next-hop)
Router(config)# ip route 192.168.2.0 255.255.255.0 10.0.0.2
#   Pakety pro 192.168.2.0/24 posílej na 10.0.0.2

# Příklad 2: Směrování přes výstupní rozhraní
Router(config)# ip route 192.168.3.0 255.255.255.0 serial0/0/0

# Výchozí trasa (default route) - 0.0.0.0/0
Router(config)# ip route 0.0.0.0 0.0.0.0 203.0.113.1

# Zobrazení routovací tabulky
Router# show ip route
# S = statická trasa, C = přímo připojená

# Smazání statické trasy
Router(config)# no ip route 192.168.2.0 255.255.255.0 10.0.0.2

3 — Statický routing v Linuxu

Linux (ip route příkazy) 
# Přidání statické trasy
$ ip route add 192.168.2.0/24 via 10.0.0.2

# Výchozí trasa
$ ip route add default via 203.0.113.1

# Přes konkrétní rozhraní
$ ip route add 192.168.3.0/24 dev eth0

# Zobrazení routovací tabulky
$ ip route show

# Smazání trasy
$ ip route del 192.168.2.0/24

# Trvalé uložení (Debian/Ubuntu): /etc/network/interfaces
# nebo (moderní): /etc/netplan/*.yaml

4 — Srovnání statického a dynamického routingu

Rozdíly 
# Statický routing:
#   + Jednoduchý, žádná režie (CPU/paměť)
#   + Větší kontrola nad toky dat
#   - Nemožnost adaptace na výpadky
#   - Náročná správa ve velkých sítích

# Dynamický routing:
#   + Automatická adaptace na výpadky
#   + Škálovatelný pro velké sítě
#   - Vyšší režie (CPU, paměť, šířka pásma)
#   - Složitější konfigurace a ladění

# Kombinace: Statický routing pro specifické cesty,
# dynamický routing pro hlavní kostru sítě
