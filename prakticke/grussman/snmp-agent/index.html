SNMP Agent

📡 SNMP 🔍 Simple Network Management Protocol

1 — Co je SNMP

SNMP (Simple Network Management Protocol) slouží k monitorování a správě síťových zařízení. Zařízení (routery, switche, servery) mají SNMP agenta, který odpovídá na dotazy správce (manager). Data jsou organizována ve stromové struktuře zvané MIB (Management Information Base).
// TIP SNMP verze: v1 (nejstarší, nezabezpečená), v2c (komunitní řetězec), v3 (šifrování a autentizace).

2 — Instalace SNMP agenta (snmpd)

Instalace na Linuxu 
# Instalace SNMP daemon a nástrojů
$ apt-get install snmp snmp-mibs-downloader snmpd

# Ověření běhu služby
$ systemctl status snmpd

# Povolení při startu
$ systemctl enable snmpd

3 — Konfigurace SNMP agenta

/etc/snmp/snmpd.conf 
# Komunitní řetězec (v2c)
rocommunity public   # read-only pro komunitu "public"
rwcommunity private  # read-write pro komunitu "private"

# Omezení přístupu na konkrétní IP
rocommunity public 192.168.0.0/24

# Systémové informace
syslocation "Server Room"
syscontact admin@firma.cz

# Poslouchat na všech rozhraních
agentAddress udp:161

# Restart po změně
$ systemctl restart snmpd

4 — Dotazování přes snmpwalk a snmpget

Základní příkazy pro dotazování 
# Procházení celého MIB stromu
$ snmpwalk -v2c -c public 192.168.0.10

# Konkrétní OID - systémový název
$ snmpget -v2c -c public 192.168.0.10 1.3.6.1.2.1.1.1.0

# Uptime systému
$ snmpget -v2c -c public 192.168.0.10 1.3.6.1.2.1.1.3.0

# Seznam rozhraní
$ snmpwalk -v2c -c public 192.168.0.10 1.3.6.1.2.1.2.2.1.2
