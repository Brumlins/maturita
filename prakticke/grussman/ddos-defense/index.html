Obrana před DDoS

🛡️ Security 💥 DDoS a Flood obrana

1 — Co je DDoS útok

DDoS (Distributed Denial of Service) je útok, který má za cíl zahltit cílový systém tolika požadavky, že přestane odpovídat legitimním uživatelům. Útoky jsou distribuované z více zdrojů.
// TIP Rozdíl mezi DoS a DDoS: DoS jde z jednoho zdroje, DDoS z tisíců různých IP adres (botnet).

2 — Typy DDoS útoků

Kategorie útoků 
# 1. Volumetrické útoky (záplava dat)
#    - UDP flood, ICMP flood
#    - Cíl: saturovat šířku pásma

# 2. Protokolové útoky (Layer 3/4)
#    - SYN flood (vyčerpání SYN tabulky)
#    - Ping of Death

# 3. Aplikační útoky (Layer 7)
#    - HTTP flood (mnoho GET/POST požadavků)
#    - Slowloris (pomalé odesílání požadavků)
#    - Cíl: vyčerpat zdroje aplikačního serveru

3 — Nástroje pro obranu

Obranné mechanismy 
# fail2ban - automatické blokování IP
$ apt-get install fail2ban

# Nastavení v /etc/fail2ban/jail.local
[apache-ddos]
enabled  = true
filter   = apache-auth
port     = http,https
logpath  = /var/log/apache2/*access.log
maxretry = 300
findtime = 600
bantime  = 3600

# Restart
$ systemctl restart fail2ban

4 — iptables pravidla proti flood útokům

Ochranná pravidla 
# Ochrana proti SYN flood
$ iptables -A INPUT -p tcp --syn -m limit --limit 1/s -j ACCEPT

# Ochrana proti ping flood (ICMP)
$ iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s -j ACCEPT

# Omezení počtu spojení z jedné IP na port 80
$ iptables -A INPUT -p tcp --dport 80 -m connlimit --connlimit-above 20 -j REJECT

# Uložení pravidel (Debian/Ubuntu)
$ iptables-save > /etc/iptables/rules.v4

# Nástroj pro analýzu provozu: iftop, nethogs
$ apt-get install iftop nethogs
