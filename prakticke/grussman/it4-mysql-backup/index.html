MySQL Backup (IT4)

🗄️ MySQL 💾 zálohování a obnova (pokročilé)

1 — Teorie zálohování databází

Zálohování MySQL může být prováděno několika způsoby: logické zálohy (mysqldump), fyzické zálohy (kopírování souborů databáze) a přírůstkové zálohy (přes binlog).
Typy záloh 
# Full backup (plná záloha) - záloha všeho
# Incremental backup (přírůstková) - jen změny od poslední zálohy
# Differential backup (rozdílová) - změny od poslední plné zálohy

# Stratégie 3-2-1:
#   3 kopie dat, 2 různá média, 1 offsite (mimo lokalitu)

# RPO (Recovery Point Objective) - max. ztráta dat (např. 1 hodina)
# RTO (Recovery Time Objective) - max. čas na obnovu (např. 4 hodiny)

2 — Pokročilé použití mysqldump

Příkazy pro zálohu 
# Konzistentní záloha InnoDB (transakce)
$ mysqldump -u root -p --single-transaction --routines --triggers --events moje_db > backup.sql

# Záloha s kompresí (lzma - lepší komprese než gzip)
$ mysqldump -u root -p --single-transaction moje_db | xz > backup.sql.xz

# Paralelní záloha (mydumper - nutno instalovat)
$ mydumper -u root -p -B moje_db -o /backups/mysql/

# Záloha uživatele (přenost práv)
$ mysqldump -u root -p --system=all > users_backup.sql

3 — Binlog - přírůstkové zálohy

Nastavení binlog 
# /etc/mysql/mysql.conf.d/mysqld.cnf
[mysqld]
server-id = 1
log_bin = /var/log/mysql/mysql-bin.log
binlog_format = ROW
expire_logs_days = 7

# Restart MySQL
$ systemctl restart mysql

# Zobrazení binlog souborů
mysql> SHOW BINARY LOGS;

# Čtení binlog (obnova konkrétních transakcí)
$ mysqlbinlog /var/log/mysql/mysql-bin.000001

# Obnova do určitého času
$ mysqlbinlog --start-datetime="2024-01-01 10:00:00" --stop-datetime="2024-01-01 12:00:00" mysql-bin.000001 | mysql -u root -p

4 — Obnova databáze (scénáře)

Postup obnovy 
# Scénář 1: Plná obnova ze SQL souboru
$ mysql -u root -p moje_db < backup.sql

# Scénář 2: Obnova + binlog (granulární)
# 1. Obnova plné zálohy
$ mysql -u root -p moje_db < full_backup.sql
# 2. Aplikace binlogů (přírůstky)
$ mysqlbinlog mysql-bin.000001 mysql-bin.000002 | mysql -u root -p moje_db

# Scénář 3: Obnova na jiný server (disaster recovery)
# 1. Zkopírovat zálohy na nový server
# 2. Nainstalovat MySQL stejné verze
# 3. Provést obnovu stejným způsobem

# Ověření integrity po obnově
mysql> CHECK TABLE uzivatele, objednavky;
