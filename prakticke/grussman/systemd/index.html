systemd

🐧 Linux ⚙️ správa daemonů a služeb

1 — Co je systemd

systemd je systémový správce (init system) pro Linux, který spravuje služby, připojování disků, síť a další systémové procesy. Nahrazuje starší SysV init. Hlavní příkaz pro správu je systemctl.

2 — Typy jednotek (Units)

Hlavní typy unit souborů 
# .service - systémová služba/daemon
# .socket - síťový socket
# .mount - připojení filesystemu
# .timer - časovač (nahrazuje cron)
# .target - skupina jednotek (např. multi-user.target)

# Umístění souborů:
# /lib/systemd/system/ - systémové služby
# /etc/systemd/system/ - uživatelské/upravené služby

3 — Vytvoření vlastní služby

/etc/systemd/system/moje-sluzba.service 
[Unit]
Description=Moje vlastní služba
After=network.target

[Service]
Type=simple
User=uzivatel
WorkingDirectory=/home/uzivatel/app
ExecStart=/usr/bin/python3 /home/uzivatel/app/server.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
Aktivace služby 
# Načtení změn v systemd
$ systemctl daemon-reload

# Povolení služby při startu
$ systemctl enable moje-sluzba

# Spuštění služby
$ systemctl start moje-sluzba

4 — Příkazy systemctl

Přehled příkazů 
# Stav služby
$ systemctl status nginx

# Spuštění / zastavení / restart
$ systemctl start nginx
$ systemctl stop nginx
$ systemctl restart nginx

# Povolení / zakázání při bootu
$ systemctl enable nginx
$ systemctl disable nginx

# Seznam všech služeb
$ systemctl list-units --type=service

# Maskování služby (úplné zakázání)
$ systemctl mask nginx
