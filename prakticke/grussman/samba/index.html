🐧 Linux 🖥️ sdílení pro Windows i Linux

Co je Samba

Samba implementuje protokol SMB/CIFS na Linuxu — sdílí soubory a tiskárny kompatibilně s Windows. Na rozdíl od NFS funguje i pro Windows klienty.

Instalace

Debian/Ubuntu 
$ apt-get install samba

# Ověření verze:
$ samba --version

Konfigurace — /etc/samba/smb.conf

Otevření konfiguračního souboru 
$ mcedit /etc/samba/smb.conf
# nebo:
$ nano /etc/samba/smb.conf
Příklad konfigurace sdílené složky 
# Přidat na konec /etc/samba/smb.conf:
[accounting]
  comment          = Sdileny adresar pro ucetnictvi
  path             = /data
  valid users      = @users       ; uživatelé ve skupině users
  writable         = yes
  force group      = users        ; nové soubory patří skupině users
  create mode      = 0660
  directory mode   = 0770
DirektivaPopis
pathcesta k sdílenému adresáři na serveru
valid userspovolení uživatelé; @skupina = všichni ve skupině
writable = yespovolení zápisu (= read only = no)
create modepráva pro nově vytvořené soubory
directory modepráva pro nově vytvořené adresáře
browseableviditelnost sdílení v síťovém okolí (yes/no)
// POZOR Nekopíruj komentáře ze vzorové konfigurace — mohou způsobit chyby při parsování. Sleduj logy: tail -f /var/log/samba/log.smb

Příprava adresáře a uživatelů

Adresář + uživatel + Samba heslo 
# Vytvoření sdíleného adresáře:
$ mkdir /data
$ chown student:users /data

# Přidání uživatele do skupiny users:
$ usermod -a -G users student

# Nastavení Samba hesla (odlišné od systémového hesla!):
$ smbpasswd -a student

# Kontrola Samba uživatelů:
$ pdbedit -L

Restart a ladění

Restart + ověření konfigurace 
# Ověření syntaxe smb.conf:
$ testparm

# Restart Samby (Ubuntu 18.04+):
$ sudo service smbd restart
# nebo:
$ systemctl restart smbd nmbd

# Sledování logů:
$ tail -f /var/log/samba/log.smb

Připojení z Windows

Ve Windows: Win+R → \\192.168.0.10\accounting → zadej Samba uživatele a heslo.

Nebo přes Průzkumník: Síť → přidat síťové umístění.
Připojení z Linux klienta 
$ apt-get install smbclient cifs-utils

# Výpis sdílení na serveru:
$ smbclient -L //192.168.0.10 -U student

# Připojení přes mount:
$ mount -t cifs //192.168.0.10/accounting /mnt/samba
      -o username=student,password=heslo
// WINDOWS 10 KOMPATIBILITA Pokud se Windows 10 nepřipojí, může být nutné snížit verzi SMB nebo zapnout SMBv1 v nastavení Windows Features.
