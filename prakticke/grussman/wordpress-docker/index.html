1 — WordPress v Dockeru

Spuštění WordPressu v Dockeru umožňuje rychlé nasazení bez nutnosti ruční instalace PHP, MySQL a webového serveru. Pomocí Docker Compose lze snadno definovat služby (WordPress + MariaDB) v jednom souboru.

2 — Docker Compose konfigurace

docker-compose.yml 
version: '3.8'

services:
  db:
    image: mariadb:latest
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: rootpass
      MYSQL_DATABASE: wordpress
      MYSQL_USER: wpuser
      MYSQL_PASSWORD: wppass
    volumes:
      - db_data:/var/lib/mysql

  wordpress:
    depends_on:
      - db
    image: wordpress:latest
    restart: always
    ports:
      - "8080:80"
    environment:
      WORDPRESS_DB_HOST: db:3306
      WORDPRESS_DB_USER: wpuser
      WORDPRESS_DB_PASSWORD: wppass
      WORDPRESS_DB_NAME: wordpress
    volumes:
      - wp_data:/var/www/html

volumes:
  db_data:
  wp_data:

3 — Instalace a první spuštění

Spuštění pomocí Docker Compose 
# Spuštění v pozadí
$ docker-compose up -d

# Zobrazení běžících služeb
$ docker-compose ps

# Přístup přes prohlížeč:
# http://localhost:8080

# Zastavení služeb
$ docker-compose down

# Zastavení a smazání dat
$ docker-compose down -v

4 — Správa WordPress kontejneru

Užitečné příkazy 
# Připojení do WordPress kontejneru
$ docker-compose exec wordpress bash

# Záloha WordPress souborů
$ docker cp wordpress_wordpress_1:/var/www/html ./backup

# Záloha databáze
$ docker-compose exec db mysqldump -u wpuser -pwppass wordpress > backup.sql

# Obnova databáze
$ docker-compose exec -T db mysql -u wpuser -pwppass wordpress < backup.sql
