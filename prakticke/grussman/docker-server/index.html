🐳 Docker ⚙️ instalace a správa

1 — Instalace Dockeru na Linuxu

Docker je platforma pro vývoj, nasazení a běh aplikací v kontejnerech. Kontejnery izolují aplikace od infrastruktury a umožňují jejich snadný přenos.
Instalace na Debian/Ubuntu 
# Aktualizace repozitářů
$ apt-get update

# Instalace potřebných balíčků
$ apt-get install ca-certificates curl gnupg lsb-release

# Přidání oficiálního GPG klíče Dockeru
$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | gpg --dearmor -o /etc/apt/trusted.gpg.d/docker.gpg

# Přidání repozitáře Dockeru
$ echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/trusted.gpg.d/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" > /etc/apt/sources.list.d/docker.list

# Instalace Docker Engine
$ apt-get update
$ apt-get install docker-ce docker-ce-cli containerd.io docker-compose-plugin
Ověření instalace 
$ docker --version
$ docker run hello-world

2 — Základní příkazy Dockeru

Přehled příkazů 
# Seznam běžících kontejnerů
$ docker ps

# Seznam všech kontejnerů (včetně zastavených)
$ docker ps -a

# Seznam stažených images
$ docker images

# Informace o Dockeru
$ docker info

# Vyčištění nepoužívaných objektů
$ docker system prune -a

3 — Práce s images

Stahování a správa images 
# Stažení image z Docker Hub
$ docker pull ubuntu:latest

# Spuštění image v kontejneru
$ docker run -it ubuntu:latest /bin/bash

# Smazání image
$ docker rmi ubuntu:latest

# Hledání images na Docker Hub
$ docker search nginx

4 — Správa kontejnerů

Životní cyklus kontejneru 
# Spuštění kontejneru na pozadí
$ docker run -d --name muj-nginx -p 8080:80 nginx

# Zastavení kontejneru
$ docker stop muj-nginx

# Spuštění zastaveného kontejneru
$ docker start muj-nginx

# Smazání kontejneru
$ docker rm muj-nginx

# Zobrazení logů kontejneru
$ docker logs muj-nginx

# Připojení do běžícího kontejneru
$ docker exec -it muj-nginx /bin/bash
