🌐 Topologie sítě
  • Rozpočítat zadaný IP rozsah pro VLANy a také propojovací podsítě (ty většinou stačí /30)
  • Vytvořit VLANy pro uživatele a přiřadit k portům, pokud bude nutné SSH na switchi, tak si udělat VLAN999 Administrative a spravce-PC by mohl být v portu s VLAN999 nebo nutnost routingu mezi VLANami
  • IP na PCs — buď pevně nastavené, nebo obdrží z DHCP služby, ale z DHCP řešte, až po funkčním routování
  • Mezi switchi propoj zrealizovat jako Link Aggregation, tedy jeden z režimů EtherChannel, nebo jindy LACP, nebo jindy PagP
  • Routování, umět obě metody statické a OSPF, zkusit si pingnout mezi zařízeními SSPU.local
  • Bude-li požadovaná DHCP, tak může být na Router1, nebo v jiném zadání na Router2
  • Bude-li požadováno SSH, tak může být na Router1, nebo v jiném zadání na Router0, nebo i třeba Switch0
  • Na hraničním routeru děláte DEFAULT ROUTE a tu buď distribuuje ostatním routerům OSPF přes doplněný příkaz, nebo u statického routování to musíte udělat Vy na všech ostatních routerech
  • NAT bude pouze na hraničním Routeru1, překlad private do veřejné jedné IP
  • PORT FORWARDING — přístup z internet serveru na služby FTP 22,23, HTTPS 443 běžící v DMZ školy na zařízení Server0
  • ACL, například z jedné VLANy je povoleno pouze https do dmz Server a z druhé VLANy pouze ftp do tohoto dmz Server0

IP plán — subnetting
FIRMA.NET: zvolte private rozsah až /22
Rozpočítejte zadaný IP rozsah pro VLANy a propojovací podsítě (ty stačí /30). Příklad níže počítá z rozsahu 10.0.0.0/22 — vyberte si vlastní private rozsah.
// IP plán — příklad s 10.0.0.0/22
Síť / VLANVelikostCIDRRozsah adresMaskaGateway
VLAN10 IT_OBOR 512 adres → /23 /23 10.0.0.0 – 10.0.1.255 255.255.254.0 10.0.0.1
VLAN20 SV_OBOR 256 adres → /24 /24 10.0.2.0 – 10.0.2.255 255.255.255.0 10.0.2.1
DMZ (Server0) 16 adres → /28 /28 10.0.3.0 – 10.0.3.15 255.255.255.240 10.0.3.1
VLAN999 Admin 16 adres → /28 /28 10.0.3.16 – 10.0.3.31 255.255.255.240 10.0.3.17
Router0 ↔ Router1 2 adresy → /30 /30 10.0.3.32 – 10.0.3.35 255.255.255.252 —
Router1 ↔ Router2 2 adresy → /30 /30 10.0.3.36 – 10.0.3.39 255.255.255.252 —
Router0 (Gi0/1 — VLAN gateway) rozhraní /23 10.0.0.1 (VLAN10), 10.0.2.1 (VLAN20), 10.0.3.17 (VLAN999) viz VLANy —
Router0 (Gi0/2 — link → Router1) rozhraní /30 10.0.3.33 255.255.255.252 —
Router2 (Gi0/0 — link → Router1) rozhraní /30 10.0.3.38 255.255.255.252 —
Router1 (Gi0/0 — inside NAT) rozhraní /30 10.0.3.34 255.255.255.252 —
Router1 (Gi0/2 — outside / Internet) rozhraní /24 213.200.13.3 255.255.255.0 —
OUTSIDE (Internet) dáno zadáním /24 213.200.13.0 – 213.200.13.255 255.255.255.0 213.200.13.1

b)
VLANy — vytvoření a přiřazení portů
Switch0, Switch1
Vytvořte VLAN10 (IT_OBOR), VLAN20 (SV_OBOR) a VLAN999 (Administrative). Přiřaďte porty k VLANám, nastavte trunk porty ke druhému switchi a routeru.
Switch0 / Switch1 
Switch> enable
Switch# configure terminal

! ── Vytvoření VLANů ─────────────────────────────────
Switch(config)# vlan 10
Switch(config-vlan)# name IT_OBOR
Switch(config-vlan)# exit
Switch(config)# vlan 20
Switch(config-vlan)# name SV_OBOR
Switch(config-vlan)# exit
Switch(config)# vlan 999
Switch(config-vlan)# name Administrative
Switch(config-vlan)# exit

! ── Access porty (PC do VLANu) ───────────────────────
Switch(config)# interface FastEthernet0/10
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
Switch(config-if)# exit

Switch(config)# interface FastEthernet0/20
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 20
Switch(config-if)# exit

! ── Trunk port (k routeru nebo druhému switchi) ───────
Switch(config)# interface GigabitEthernet0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan all
Switch(config-if)# exit

! ── Management IP na VLAN999 (pro SSH na switch) ──────
Switch(config)# interface vlan 999
Switch(config-if)# ip address 10.0.3.18 255.255.255.240
Switch(config-if)# no shutdown
Switch(config-if)# exit
Switch(config)# ip default-gateway 10.0.3.17
// VYSVĚTLENÍ
  • switchport mode access – port pro jediný VLAN (PC)
  • switchport mode trunk – port nese více VLANů (k routeru / switchi)
  • interface vlan 999 – L3 rozhraní VLANu pro management přístup
  • ip default-gateway – nutné na switchi (L2) pro dosažitelnost mimo lokální síť

Link Aggregation — EtherChannel / LACP / PagP
Switch0 ↔ Switch1
Propoj mezi switchci realizuj jako Link Aggregation. Zadání specifikuje jeden režim: EtherChannel (ON), nebo LACP, nebo PagP.
LACP — Switch0 (active) + Switch1 (passive) 
! ── Switch0 (active) ────────────────────────────────
Switch0(config)# interface range FastEthernet0/1-3
Switch0(config-if-range)# channel-protocol lacp
Switch0(config-if-range)# channel-group 1 mode active
Switch0(config-if-range)# exit
Switch0(config)# interface Port-channel1
Switch0(config-if)# switchport mode trunk

! ── Switch1 (passive) ───────────────────────────────
Switch1(config)# interface range FastEthernet0/1-3
Switch1(config-if-range)# channel-protocol lacp
Switch1(config-if-range)# channel-group 1 mode passive
Switch1(config-if-range)# exit
Switch1(config)# interface Port-channel1
Switch1(config-if)# switchport mode trunk
PagP — Switch0 (desirable) + Switch1 (auto) 
Switch0(config)# interface range FastEthernet0/1-3
Switch0(config-if-range)# channel-group 1 mode desirable

Switch1(config)# interface range FastEthernet0/1-3
Switch1(config-if-range)# channel-group 1 mode auto
EtherChannel ON — oba switche stejně 
Switch(config)# interface range FastEthernet0/1-3
Switch(config-if-range)# channel-group 1 mode on
Switch(config-if-range)# exit
Switch(config)# interface Port-channel1
Switch(config-if)# switchport mode trunk

! Ověření:
Switch# show etherchannel summary
// VYSVĚTLENÍ
  • LACP – IEEE standard, active/active nebo active/passive
  • PagP – Cisco proprietární, desirable/desirable nebo desirable/auto
  • ON – bez vyjednávání, oba musí být ON
  • Port-channel1 musí být nastaven jako trunk, pokud nese více VLANů
e)
Routing — inter-VLAN + mezi routery
Router0, Router1, Router2 — statický i OSPF
Nastavte inter-VLAN routing (router-on-a-stick na Router0) a routování mezi routery metodou statickou i OSPF. Ověřte ping mezi zařízeními v rámci SSPU.local.
Router0 — inter-VLAN (router-on-a-stick) 
Router0(config)# interface GigabitEthernet0/1
Router0(config-if)# no shutdown
Router0(config-if)# exit

! Sub-interface pro VLAN10
Router0(config)# interface GigabitEthernet0/1.10
Router0(config-subif)# encapsulation dot1Q 10
Router0(config-subif)# ip address 10.0.0.1 255.255.254.0
Router0(config-subif)# exit

! Sub-interface pro VLAN20
Router0(config)# interface GigabitEthernet0/1.20
Router0(config-subif)# encapsulation dot1Q 20
Router0(config-subif)# ip address 10.0.2.1 255.255.255.0
Router0(config-subif)# exit

! Sub-interface pro VLAN999
Router0(config)# interface GigabitEthernet0/1.999
Router0(config-subif)# encapsulation dot1Q 999
Router0(config-subif)# ip address 10.0.3.17 255.255.255.240
Router0(config-subif)# exit

! Propojovací linka Router0 ↔ Router2
Router0(config)# interface GigabitEthernet0/2
Router0(config-if)# ip address 10.0.3.41 255.255.255.252
Router0(config-if)# no shutdown
OSPF — všechny routery (area 0) 
! Na každém routeru: přidejte všechny přímo připojené sítě (Router0):
Router(config)# router ospf 1
Router(config-router)# network 10.0.3.32 0.0.0.3 area 0
Router(config-router)# network 10.0.2.40 0.0.0.3 area 0
Router(config-router)# network 10.0.0.0 0.0.3.255  area 0
Router(config-router)# exit

! Ověření:
Router# show ip route ospf
Router# show ip ospf neighbor
Statické routování (alternativa k OSPF) 
! Na Router0: trasy ke všem vzdáleným sítím
Router0(config)# ip route 10.0.0.0 255.255.254.0 10.0.3.34
Router0(config)# ip route 10.0.2.0 255.255.255.0 10.0.3.34
Router0(config)# ip route 10.0.3.0  255.255.255.240 10.0.3.34
// POZOR Wildcard maska v OSPF je inverzní k síťové masce: /23 = 255.255.254.0 → wildcard 0.0.1.255

DHCP server
Router1 (nebo Router2 dle zadání)
Nakonfigurujte DHCP pro VLAN10 a VLAN20. Řešte až po funkčním routování. Pokud je DHCP server jiný router než ten, kde jsou sub-interfaces, přidejte ip helper-address.
Router1 — DHCP pooly 
! Vyřaďte gateway adresy z distribuce
Router1(config)# ip dhcp excluded-address 10.0.0.1 10.0.0.10
Router1(config)# ip dhcp excluded-address 10.0.2.1 10.0.2.10

! Pool pro VLAN10
Router1(config)# ip dhcp pool VLAN10
Router1(dhcp-config)# network 10.0.0.0 255.255.254.0
Router1(dhcp-config)# default-router 10.0.0.1
Router1(dhcp-config)# exit

! Pool pro VLAN20
Router1(config)# ip dhcp pool VLAN20
Router1(dhcp-config)# network 10.0.2.0 255.255.255.0
Router1(dhcp-config)# default-router 10.0.2.1
Router1(dhcp-config)# exit

! Pokud je DHCP na jiném routeru než gateway — helper-address
Router2(config)# interface GigabitEthernet0/1.10
Router2(config-subif)# ip helper-address 10.0.3.42
g)
SSH přístup
Router1 (nebo Router0 / Switch0 dle zadání)
Povolte SSH přístup na zařízení. Vyžaduje hostname, domain-name a vygenerování RSA klíče. Omezte přístup ACL — pouze z VLAN999 (Administrative).
Router1 — SSH konfigurace 
Router1(config)# hostname Router1
Router1(config)# ip domain-name sspu.local
Router1(config)# crypto key generate rsa modulus 1024
Router1(config)# ip ssh version 2

! Lokální uživatel
Router1(config)# username admin privilege 15 secret heslo

! VTY liny — pouze SSH, lokální login
Router1(config)# line vty 0 15
Router1(config-line)# login local
Router1(config-line)# transport input ssh
Router1(config-line)# exit

! ACL — SSH přístup pouze z VLAN999
Router1(config)# ip access-list standard SSH_ONLY
Router1(config-std-nacl)# permit 10.0.3.16 0.0.0.15
Router1(config-std-nacl)# exit
Router1(config)# line vty 0 15
Router1(config-line)# access-class SSH_ONLY in
// OVĚŘENÍ Z PC v VLAN999: PC → Desktop → Command Prompt → ssh -l admin 10.0.3.X
h)
Default route — hraniční router
Router1 (hraniční) → distribuce OSPF nebo staticky
Na hraničním routeru nastavte default route do Internetu. Ostatní routery ji získají buď přes OSPF redistribuci, nebo ji nastavte ručně.
Router1 — default route + OSPF redistribuce 
! Default route na Internet
Router1(config)# ip route 0.0.0.0 0.0.0.0 213.200.13.2

! Distribuce default route přes OSPF (ostatní routery ji dostanou automaticky)
Router1(config)# router ospf 1
Router1(config-router)# default-information originate
Router1(config-router)# exit

! Pokud používáte statické routování — na ostatních routerech ručně:
Router0(config)# ip route 0.0.0.0 0.0.0.0 10.0.3.33
Router2(config)# ip route 0.0.0.0 0.0.0.0 10.0.3.37
i)
NAT — překlad private → veřejná IP
Router1 (hraniční)
NAT pouze na hraničním Routeru1. Překlad všech privátních adres na jednu veřejnou IP (PAT / NAT overload).
Router1 — NAT overload (PAT) 
! Označení vnitřního rozhraní
Router1(config)# interface GigabitEthernet0/0
Router1(config-if)# ip nat inside
Router1(config-if)# exit

! Označení vnějšího rozhraní (k Internetu)
Router1(config)# interface GigabitEthernet0/2
Router1(config-if)# ip nat outside
Router1(config-if)# exit

! ACL — co se překládá (privátní rozsah)
Router1(config)# ip access-list standard NAT_INSIDE
Router1(config-std-nacl)# permit 10.0.0.0 0.0.3.255
Router1(config-std-nacl)# exit

! PAT — překlad na jednu veřejnou IP
Router1(config)# ip nat inside source list NAT_INSIDE
              interface GigabitEthernet0/1 overload

! Ověření:
Router1# show ip nat translations
// POZOR Nezapomeňte na ip nat inside na VŠECH vnitřních rozhraních (i sub-interfaces na Router2 pokud routujete přes něj).
j)
Port Forwarding — přístup z Internetu do DMZ
Router1 — FTP (20,21) + HTTPS (443) → Server0
Zpřístupněte ze strany Internetu služby na Server0 v DMZ: FTP (port 20 a 21) a HTTPS (port 443).
Router1 — statický NAT (port forwarding) 
! HTTPS 443 → Server0
Router1(config)# ip nat inside source static tcp
              10.0.3.2 443  213.200.13.X 443

! FTP 21 (řídící kanál) → Server0
Router1(config)# ip nat inside source static tcp
              10.0.3.2 21   213.200.13.X 21

! FTP 20 (datový kanál) → Server0
Router1(config)# ip nat inside source static tcp
              10.0.3.2 20   213.200.13.X 20

! HTTP 80 (pokud požadováno)
Router1(config)# ip nat inside source static tcp
              10.0.3.2 80   213.200.13.X 80
// TIP 213.200.13.X = veřejná IP adresy Router1 na rozhraní do Internetu (GigabitEthernet0/1). Server0 musí mít v GUI povolené požadované služby (http, https, ftp).
k)
ACL — řízení přístupu do DMZ
VLAN10 → pouze HTTPS · VLAN20 → pouze FTP
Z VLAN10 povolte pouze HTTPS (443) na Server0 v DMZ. Z VLAN20 povolte pouze FTP (20, 21) na Server0 v DMZ. Veškerý ostatní provoz do DMZ zakažte.
Router2 — Extended ACL aplikovaná na sub-interfaces 
! ACL pro VLAN10 — pouze HTTPS do DMZ
Router2(config)# ip access-list extended VLAN10_DMZ
Router2(config-ext-nacl)# permit tcp 10.0.0.0 0.0.1.255 host 10.0.3.2 eq 443
Router2(config-ext-nacl)# permit ip  10.0.0.0 0.0.1.255 10.0.0.0 0.0.3.255
Router2(config-ext-nacl)# deny   ip  any any
Router2(config-ext-nacl)# exit

! ACL pro VLAN20 — pouze FTP do DMZ
Router2(config)# ip access-list extended VLAN20_DMZ
Router2(config-ext-nacl)# permit tcp 10.0.2.0 0.0.0.255 host 10.0.3.2 eq 21
Router2(config-ext-nacl)# permit tcp 10.0.2.0 0.0.0.255 host 10.0.3.2 eq 20
Router2(config-ext-nacl)# permit ip  10.0.2.0 0.0.0.255 10.0.2.0 0.0.0.255
Router2(config-ext-nacl)# deny   ip  any any
Router2(config-ext-nacl)# exit

! Aplikace ACL na vstupní sub-interfaces
Router2(config)# interface GigabitEthernet0/0.10
Router2(config-subif)# ip access-group VLAN10_DMZ in
Router2(config-subif)# exit

Router2(config)# interface GigabitEthernet0/0.20
Router2(config-subif)# ip access-group VLAN20_DMZ in
// VYSVĚTLENÍ
  • in – ACL filtruje provoz přicházející do tohoto rozhraní (od PCček)
  • Druhý permit ip povoluje komunikaci uvnitř VLANu (vzájemně mezi PC)
  • Poslední deny ip any any je implicitní, ale přidáním ji vidíte v show access-list
