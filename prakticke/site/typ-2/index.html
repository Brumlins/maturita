🌐 Topologie sítě
  • Rozpočítat zadaný IP rozsah pro VLANy a také propojovací podsítě (ty většinou stačí /30)
  • Vytvořit VLANy pro uživatele a přiřadit k portům, pokud bude nutné SSH na switchi, tak si udělat VLAN999 Administrative a spravce-PC by mohl být v portu s VLAN999 nebo nutnost routingu mezi VLANami
  • IP na PCs — buď pevně nastavené, nebo obdrží z DHCP služby, ale z DHCP řešte, až po funkčním routování
  • Mezi switchi propoj zrealizovat jako Link Aggregation, tedy jeden z režimů EtherChannel, nebo jindy LACP, nebo jindy PagP
  • Routování, umět obě metody statické a OSPF, zkusit si pingnout mezi zařízeními COMPANY.local
  • Bude-li požadovaná DHCP, tak může být na Router1, nebo v jiném zadání na Router2
  • Bude-li požadováno SSH, tak může být na Router1, nebo v jiném zadání na Router0, nebo i třeba Switch0
  • Na hraničním routeru děláte DEFAULT ROUTE a tu buď distribuuje ostatním routerům OSPF přes doplněný příkaz, nebo u statického routování to musíte udělat Vy na všech ostatních routerech
  • NAT bude pouze na hraničním Routeru1, překlad private do veřejné jedné IP
  • PORT FORWARDING — přístup z internet serveru na služby FTP 22,23, HTTPS 443 běžící v DMZ školy na zařízení Server0
  • ACL, například z jedné VLANy je povoleno pouze https do dmz Server a z druhé VLANy pouze ftp do tohoto dmz Server0

VLANy — vytvoření a přiřazení portů na switchi
SW1, SW2 — VLAN100, VLAN200, VLAN900
Vytvořte VLAN100, VLAN200 a VLAN900 (DMZ) na obou switchi. Přiřaďte access porty k VLANům a nastavte trunk porty mezi SW1 ↔ SW2. Mezi switch a routerem není žádné přímé propojení — R2 používá gateway z VLAN900 (10.1.2.1).
SW1 / SW2 — VLAN creation + port assignment 
SW1> enable
SW1# configure terminal

! ── Vytvoření VLANů ─────────────────────────────────

SW1(config-vlan)# name VLAN100
SW1(config-vlan)# exit
SW1(config)# vlan 200
SW1(config-vlan)# name VLAN200
SW1(config-vlan)# exit
SW1(config)# vlan 900
SW1(config-vlan)# name DMZ
SW1(config-vlan)# exit

! ── Access porty (PC do VLANu) ───────────────────────
SW1(config)# interface range FastEthernet0/1-3
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 100
SW1(config-if-range)# exit

SW1(config)# interface range FastEthernet0/4-6
SW1(config-if-range)# switchport mode access
SW1(config-if-range)# switchport access vlan 200
SW1(config-if-range)# exit

! ─ Trunk port (SW1 ↔ SW2) ───────────────────────────
SW1(config)# interface FastEthernet0/24
SW1(config-if)# switchport mode trunk
SW1(config-if)# switchport trunk allowed vlan all
SW1(config-if)# exit

  • interface range Fa0/4-6 — access porty pro PC ve VLAN200

Trunk — SW1 → Router2 (VLAN900)
Žádné přímé propojení — R2 používá VLAN900 gateway 10.1.2.1
Mezi switch a routerem není žádné přímé propojení. Router2 používá gateway z VLAN900 → 10.1.2.1. Servery v DMZ (https, ftp-log) mají gateway 10.1.2.1. Konfigurujte sub-interface na R2 pro inter-VLAN routing přes VLAN900.
R2 — inter-VLAN routing přes VLAN900 
! ── Sub-interface pro VLAN100 ────────────────────────
R2(config)# interface GigabitEthernet0/0.100
R2(config-subif)# encapsulation dot1Q 100
R2(config-subif)# ip address 10.1.0.1 255.255.255.0
R2(config-subif)# exit

! ── Sub-interface pro VLAN200 ────────────────────────
R2(config)# interface GigabitEthernet0/0.200
R2(config-subif)# encapsulation dot1Q 200
R2(config-subif)# ip address 10.1.1.1 255.255.255.0
R2(config-subif)# ip helper-address 10.1.2.65
R2(config-subif)# exit

! ── Sub-interface pro VLAN900 (DMZ) — gateway pro servery ──
R2(config)# interface GigabitEthernet0/0.900
R2(config-subif)# encapsulation dot1Q 900
R2(config-subif)# ip address 10.1.2.1 255.255.255.192
R2(config-subif)# no shutdown
R2(config-subif)# exit
// POZOR Mezi SW1 a R2 není žádné přímé kabelové propojení. R2 routuje mezi VLANy přes sub-interfaces. VLAN900 slouží jako DMZ — servery mají gateway 10.1.2.1.

Link Aggregation — SW1 ↔ SW2
LACP / PagP / EtherChannel dle zadání
Realizujte Link Aggregation mezi SW1 a SW2. Port range: FastEthernet0/1-3. Port-channel nastavte jako trunk.
LACP — SW1 (active) + SW2 (passive) 
! SW1

SW1(config-if-range)# channel-protocol lacp
SW1(config-if-range)# channel-group 1 mode active

SW1(config)# interface Port-channel1

! SW2
SW2(config)# interface range FastEthernet0/1-3

SW2(config-if)# switchport mode trunk

NAT + Default route — R1
Pouze VLAN100 má přístup do Internetu · next hop 213.200.13.2
Na R1 nakonfigurujte NAT overload — přístup do Internetu pouze pro zařízení z VLAN100. Z VLAN200 přístup do Internetu není povolen. Nezapomeňte na default route (next hop = 213.200.13.2) a její distribuci k R2.

R1(config)# ip route 0.0.0.0 0.0.0.0 213.200.13.2

! OSPF — distribuce default route k R2
R1(config)# router ospf 1
R1(config-router)# default-information originate
R1(config-router)# exit

! Označení rozhraní

R1(config-if)# ip nat inside
R1(config-if)# exit
R1(config)# interface GigabitEthernet0/1
R1(config-if)# ip nat outside
R1(config-if)# exit

n)
ACL — extended ACL na R2 sub-interfaces
VLAN100 → HTTPS (443) na 10.1.2.10 · VLAN200 → deny HTTPS
Z VLAN100 povolte přístup na https server (10.1.2.10, port 443). Z VLAN200 komunikaci na https server úplně zakažte. Ostatní provoz z obou VLANů povolte.
R2 — Extended ACL (HTTPS) 
! Extended ACL — HTTPS přístup
R2(config)# ip access-list extended HTTPS
R2(config-ext-nacl)# permit tcp 10.1.0.0 0.0.0.255 host 10.1.2.10 eq 443
R2(config-ext-nacl)# deny   tcp 10.1.1.0 0.0.0.255 host 10.1.2.10 eq 443
R2(config-ext-nacl)# permit ip  10.1.0.0 0.0.0.255 any
R2(config-ext-nacl)# permit ip  10.1.1.0 0.0.0.255 any
R2(config-ext-nacl)# exit

! Aplikace na výstupní interface (směrem k DMZ)
R2(config)# interface GigabitEthernet0/1
R2(config-if)# ip access-group HTTPS out
R2(config-if)# exit

! Ověření:
R2# show access-lists
R2# show ip interface GigabitEthernet0/1
// VYSVĚTLENÍ
  • permit tcp 10.1.0.0/24 → host 10.1.2.10 eq 443 — VLAN100 může na HTTPS
  • deny tcp 10.1.1.0/24 → host 10.1.2.10 eq 443 — VLAN200 nesmí na HTTPS
  • permit ip ... any — ostatní provoz z obou VLANů povolen
  • ACL aplikováno out na Gi0/1 (směr k R1/DMZ)
  • deny pravidlo musí být před obecným permit ip
